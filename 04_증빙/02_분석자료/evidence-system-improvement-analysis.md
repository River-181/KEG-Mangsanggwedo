## 2. 엑셀 트래커 분석 (KEG_AI_Prompt_Tracker.xlsx)

### 엑셀 스키마 (5개 시트)

| 시트 | 컬럼 수 | 핵심 내용 |
|------|---------|----------|
| Prompt_Log | 40 | 환경, 모델, 버전, 티어, 사고수준, 비용, 민감도까지 포괄 |
| Session_Log | 15 | 세션 단위 맥락/목표/결과 |
| Artifact_Log | 9 | 프롬프트↔산출물 연결 (P-001 → A-001) |
| Tool_Log | 18 | 도구별 메타 (강점/약점/언제 쓰나) |
| Phase_Map | 9 | 단계별 진행 상태 |

### 이미 기록된 데이터 (ChatGPT Pro 세션)

- P-001~P-004: 대회 리서치, 전략 수립, 운영 문서화, 트래커 생성
- S-001: 첫 세션 (대회 개요 정리)
- A-001~A-004: 개요서, 트래커, 홍보PDF 2종
- T-001~T-003: ChatGPT Web, Codex, Perplexity

### 엑셀의 강점 vs 한계

| 강점 | 한계 |
|------|------|
| 40컬럼 정밀 스키마 | AI 에이전트가 .xlsx append 불가 |
| 프롬프트↔산출물 연결 | git diff 불가 |
| 도구 메타 관리 | 옵시디언 검색/링크 불가 |
| 단계별 추적 | 2명 동시 편집 불가 |

### 결론: 엑셀 스키마를 마크다운으로 전환

엑셀의 스키마는 **설계 문서로 보존** (`assets/originals/`).
실제 기록은 AI가 자동 append 가능한 마크다운 테이블로.

현재 `ai-usage-log.md` 8컬럼 → 엑셀 수준으로 확장:
```
| ID | 시각 | Phase | 역할 | 작업 | 환경 | 클라이언트 | 모델 | 입력 | 출력 | 채택 | 이유 | Artifact | 비용구간 |
```

추가 신규 로그:
- `session-log.md` — 세션 단위
- `tool-log.md` — 도구 메타
- `evolution-log.md` — 시스템 진화 기록
- `token-strategy.md` — 토큰/비용 최적화

## 3. AI 리포트 핵심 전략: 생태계 아키텍처

### 핵심 통찰

> 500팀 중 대부분은 "ChatGPT로 코드 짰습니다"를 쓸 것이다.
> 우리는 **멀티 에이전트 협업 시스템의 설계·운영·진화 과정 자체를 보여준다.**

### 생태계 아키텍처 다이어그램

```
┌─────────────────────────────────────────────────────┐
│                AI 생태계 아키텍처                      │

│  ┌─────────┐    ┌──────────┐    ┌─────────────┐      │
│  │ 인간 2명  │───▶│ 하네스    │───▶│ AI 에이전트  │      │
│  │(승보+주용)│    │ (Rules,  │    │ (8 역할)     │      │
│  └─────────┘    │  Memory, │    └──────┬──────┘      │
│       ▲          │  Context)│           │             │
│       │          └──────────┘           ▼             │
```

### 리포트에 보여줄 것

| 주제 | 기록 소스 | 시각화 방법 |
|------|----------|------------|
| 인풋→프로세스→아웃풋 | ai-usage-log, session-log | 데이터 흐름도 |
| 도구 활용 방법 | tool-log | 도구별 매트릭스 |
| 메모리 관리 | .context/memory/ 변경 이력 | 진화 다이어그램 |
| 프롬프트 전략 | prompt-catalog | 프롬프트 분류 트리 |
| 하네스 설계 | CLAUDE.md, agents/, rules/ | 레이어 아키텍처 |
| 다중 AI 로테이션 | session-log (환경/모델 컬럼) | 모델별 역할 맵 |
| 토큰 최적화 | token-strategy | 비용 절감 그래프 |
| CLI/MCP/Skills | tool-log, plugins.md | 통합 생태계 맵 |
| 시스템 진화 | evolution-log | 타임라인 |

### 학생 예산 제약 → 강점 전환

"학생이라 토큰이 부족하다"는 약점이 아니라 **심사 어필 포인트**:
1. **모델 로테이션** — Opus(깊은 사고) / Sonnet(실행) / Haiku(로깅) 분리
2. **컨텍스트 절약** — `.context/` 공유 메모리로 반복 설명 제거
3. **하네스 설계** — CLAUDE.md + agents/ + rules/로 프롬프팅 비용 절감
4. **MCP/Skills** — 외부 도구 위임으로 토큰 소모 줄임
