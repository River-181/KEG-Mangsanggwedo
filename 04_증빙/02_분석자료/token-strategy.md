# 토큰/비용 최적화 전략

> 학생 예산 제약 하에서 다중 AI를 효율적으로 운용하는 전략을 기록한다.
> AI 리포트에서 "제한된 자원으로 최대 효과"를 어필하는 핵심 소스.

## 모델 로테이션 전략

| 모델 | 비용 | 용도 | 언제 쓰나 |
|------|------|------|----------|
| Claude Opus 4.6 | 높음 | 깊은 추론, 아키텍처 | 구조 설계, 심사 시뮬, 복잡한 분석 |
| Claude Sonnet | 중간 | 실행, 코딩 | 코드 생성, 문서 작성, 리서치 |
| Claude Haiku | 낮음 | 경량 작업 | 로깅, 간단 변환, 증빙 정리 |
| GPT-5.4 (Pro) | 정액 | 전략 설계 | 기획, 플레이북, 방향 설정 |
| GPT-5 (Codex) | 정액 | 구현/평가 | 로컬 파일, 검증, 평가 |
| Perplexity (Pro) | 정액 | 리서치 | 사례 수집, 경쟁 분석 |

## 컨텍스트 절약 기법

| 기법 | 설명 | 절감 효과 |
|------|------|----------|
| `.context/memory/` | 공유 메모리로 반복 설명 제거 | 매 세션 ~2K 토큰 절약 |
| CLAUDE.md + rules/ | 하네스로 행동 사전 규정 | 프롬프트 길이 단축 |
| PLAN.md + PROGRESS.md | 상태 동기화로 재설명 제거 | 세션 간 맥락 유지 |
| 에이전트 역할 분리 | 전문화된 지시로 정확도 향상 | 재시도 횟수 감소 |
| MCP 서버 | 외부 도구 위임으로 토큰 절약 | API 호출 대신 도구 사용 |
