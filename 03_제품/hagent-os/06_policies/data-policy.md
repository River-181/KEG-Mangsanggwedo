# Policy: 데이터 정책

> 데이터 수집·보존·외부 전송·주권·감사 기준

---

## 핵심 원칙

**1. 최소 필요 원칙**
- 교육 목적에 필요한 데이터만 수집
- 불필요한 데이터 수집 금지

**2. 로컬 우선**
- 민감한 데이터는 로컬 저장 기본 (학원 내부 서버)
- 외부 전송 최소화 (API 호출 시에만)

**3. 투명성**
- 학부모에게 어떤 데이터를 수집·사용하는지 명시
- AI가 처리하는 데이터 명시

**4. 통제**
- 학부모는 언제든 데이터 삭제 요청 가능 (30일 내 처리)
- 원장은 데이터 전송 중지/삭제 언제든 가능

---

## 수집 데이터 범위

### A. 학생 데이터 (필수 수집)
```
기본 정보:
  - 이름, 생년월일, 학년, 성별
  - 학교명, 지역

수강 정보:
  - 수강과목, 수강시간, 등록일, 종료일
  - 강사명, 강의실
  - 월 수강료, 결제 방법

학습 기록:
  - 출석/결석/지각 기록
  - 성적/진도율 (비율만 저장, 절대값 미기록)
  - 숙제 제출 여부

건강/안전 (조건부):
  - 긴급 연락처
  - 알레르기 정보 (보호자 명시 동의 필수)
  - 장애·기저질환 (맞춤형 수업용 - 동의 필수)
```

### B. 학부모 데이터 (필수 수집)
```
기본 정보:
  - 이름, 관계 (부/모/보호자)
  - 전화번호 (1개 필수)
  - 이메일 (선택, 영수증용)

결제 정보:
  - 카드번호 (일부만 저장: ****-****-1234)
  - 자동이체 계좌 (일부만 저장: 농협-***-12345)

연락 기록:
  - 로그인 기록 (학부모 앱 이용 현황)
  - 메시지 발송 기록 (감사용)
```

### C. 강사/직원 데이터 (필수 수집)
```
기본 정보:
  - 이름, 주민번호 (급여 처리용 - 암호화)
  - 전화, 주소
  - 은행계좌 (일부만: ***-12345)

근무 정보:
  - 수강 클래스 및 일정
  - 시급/월급
  - 근로계약 시작일

성과 정보:
  - 학생 만족도 (설문 결과)
  - 출석률
  - 교체/정지 기록
```

### D. 운영 데이터 (필수 수집)
```
학원 정보:
  - 학원명, 주소, 연락처, 사업자등록번호
  - 개원일, 강사 수, 학생 수

재무 정보:
  - 월 수강료 총액, 환불액, 순매출
  - 세금 (부가세, 원천징수)
  - 강사료 총액

규제 정보:
  - 소방 검사 기록
  - 노무 관련 (4대보험, 근로계약)
  - 개인정보 처리 기록
```

## 외부 전송 제한 (Claude API 등)

### 기본 원칙: 최소화 + 익명화

**절대 전송 금지 데이터:**
```
❌ 학생 주민번호
❌ 강사 급여 (실수치)
❌ 학부모 계좌번호 (완전)
❌ 의료 정보 (알레르기 등 세부)
❌ 가정 환경 정보 (부모 직업, 이혼 여부 등)
```

**조건부 전송 가능 데이터:**
```
✅ 민원 텍스트 → 개인정보 제거 후 전송
   "김민준 학부모: 환불해달라고..."
   →
   "학부모 A: 환불해달라고..."

✅ 학생 성적 통계 → 익명화 후
   "초4 평균 70점, 범위 50~92점" (OK)
   "김민준은 70점, 성적 향상 0" (NO)

✅ 강사 성과 평가 → 집계 수치만
   "평균 만족도 4.2/5" (OK)
   "박수진 강사 만족도 3.5" (NO)
```

### 전송 프로토콜

**1. 민원 분류 (Complaint Agent)**
```
[원본]
"김민준 어머니: 우리 아이가 문제가 많다고 해요..."

↓ 익명화

[Claude API 전송]
{
  "text": "학부모가 학생의 문제 행동에 대해 민원...",
  "category_hint": "behavioral_concern",
  "student_grade": "4",
  "anonymized": true
}

↓ 결과 받기

{
  "category": "behavior_issue",
  "urgency": "medium",
  "suggested_response": "학생 평가 후 학부모 상담 제안"
}

[원장 승인 후 실행]
```

**2. 이탈 감지 (Retention Agent)**
```
[원본 데이터]
학생 ID | 이름 | 로그인 | 성적 | 출석
-------
1234    | 김민준 | 0일 | 70 | 80%

↓ 익명화

[전송]
{
  "student_id": "anon_1234",
  "days_since_login": 10,
  "recent_performance": "declining",
  "attendance_rate": 0.8,
  "anonymized": true
}
```

### 데이터 로깅

모든 외부 전송 기록:
```
timestamp | agent | data_summary | anonymized | result | duration
2026-04-09T10:23:45 | Complaint | 1건 민원 | YES | ✅ | 2.3s
2026-04-09T10:45:12 | Retention | 150명 분석 | YES | ✅ | 8.5s
```

```
  - 라이선스: Apache 2.0 또는 GNU GPL 3.0
```

## 감사 로그 불변성 (Immutable Audit Log)

### 기본 구조
```
모든 데이터 접근/수정을 시간순 기록:

{
  timestamp: "2026-04-09T10:23:45Z",
  actor: "Complaint-Agent" | "Academy-Head" | "System",
  action: "READ" | "WRITE" | "DELETE" | "EXPORT",
  table: "students" | "complaints" | "payments",
  record_id: "student_1234",
  before: { ... },      # 수정 전
  after: { ... },       # 수정 후
  reason: "민원 분류",
  hash: "sha256(...)"   # 위변조 감지용
}
```

### 삭제 불가 원칙
```
✅ 삭제 요청 즉시 soft delete 실행
❌ 실제 hard delete는 금지 (대신 30일 grace period)

[즉시 처리 — soft delete]
deleted_at: "2026-04-09T10:00:00Z"
deleted_by: "parent_request"
reason: "학부모 개인정보보호법 행사"
is_soft_deleted: true
audit_log_reference: "log_2026_04_09_deleted_records"

→ 데이터 비활성화, 사용자 접근 불가

[30일 후 자동 실행 — hard delete]
grace_period_days: 30
hard_delete_scheduled: "2026-05-09T10:00:00Z"
purge_policy: "완전 삭제 (감사 로그 제외)"

→ 감사 로그는 10년 보관 (법적 증거)
```

### 감사 접근
```
원장 권한:
  [대시보드] → [감사 로그] → 검색 및 필터
  - 기간별 (2026년 1월 ~ 3월)
  - 에이전트별 (Complaint, Retention, ...)
  - 대상별 (학생, 결제, 규제)
  - 거래별 (READ/WRITE/DELETE)

외부 감시:
  - 교육청 (의무 제출 시 감사 로그 증거로 제출)
  - 개인정보보호위원회 (침해 사건 시 증거)
```

## 엑셀 Import 시 개인정보 처리

### Import 플로우
```
원장이 학원관리 엑셀 다운로드 (학원장 ERP)

↓

HagentOS로 Import

↓ 위험 데이터 감지

[경고]
"⚠️ 다음 컬럼은 외부 전송 금지입니다:
  - 주민번호
  - 부모 계좌번호 (완전)
  
제거하고 import하시겠습니까?"

↓ 원장 확인

[Import 실행 + 감사 기록]
```

### 자동 정제 규칙
```
컬럼 | 자동 처리
-----|----------
주민번호 | ❌ 거절 (수동 제거 필수)
계좌번호 | ✅ 일부만 저장 (****-12345)
전화번호 | ✅ 모두 저장 (필수)
이름 | ✅ 모두 저장 (필수)
성적 | ✅ 모두 저장 (교육용)
급여 | ❌ 거절 (급여 시스템과 별도 관리)
```
