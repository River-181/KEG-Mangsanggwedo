# Policy: 커뮤니케이션 정책

> AI가 생성한 메시지의 발송 원칙과 채널별·대상별 기준

---

## 핵심 원칙

**1. 사람 승인 우선 (Default: Level 1)**
- **기본**: AI가 생성한 모든 메시지는 원장 또는 담당자 승인 후 발송
- **예외**: Level 0 사전 승인 메시지는 자동 발송 (명시 명단 참고)

**2. 발송자 투명성**
- 모든 메시지는 "AI 생성 메시지" 표시 의무 (법적 투명성)
- 단, 학부모 경험을 고려해 표시 위치·방식 탄력적 (정책 참고)

**3. 개인정보 보호**
- 학생·학부모 이름·연락처·성적 등 민감 정보 외부 발송 금지
- 학부모 동의 필수 (마케팅 메시지)

## 채널별 사용 기준

### 앱 알림 (In-App Notification)
```
목적: 원장/담당자의 액션 필요
예시:
  - 민원 승인 대기
  - 이탈 학생 개입 제안
  - 법정 기한 알림

SLA: 5분 이내 발송
재시도: 24시간 내 (미확인 시 2회)
개인정보: ✅ 학생 이름, 금액 등 포함 가능
```

### 카카오톡 (또는 WhatsApp 등 사용자 선택)
```
목적: 학부모/학생 연락
예시:
  - 수강료 연체 안내
  - 수강료 결제 완료 알림
  - 강사 변경 공지
  - 이벤트/공지사항

SLA: 실시간 ~ 1시간 내
개인정보: ✅ 포함 (동의 확인 후)
승인 레벨:
  - Level 0 (자동): 템플릿 고정 메시지
  - Level 1 (승인): 맞춤형 메시지, 금액 관련

AI 표시: "[AI]" 또는 "시스템 메시지" (선택)
```

### SMS
```
목적: 긴급/필수 전달
예시:
  - 수강료 미납 최종 알림 (24시간 내)
  - 긴급 일정 변경 (강사 부상, 자연재해)
  - OTP 인증 (비밀번호 변경 등)

SLA: 2분 이내
개인정보: ✅ 포함 가능
승인 레벨: Level 1 필수 (비용 + 법적 책임)
AI 표시: "AI" 표시 선택사항 (긴급성 고려)
```

### 이메일
```
목적: 기록·공식 통보
예시:
  - 계약/약관 변경 공지
  - 영수증 발송
  - 보증서/교육 이수증
  - 폐원 및 법적 통보

SLA: 당일 또는 예약 발송
개인정보: ✅ 포함 (암호화 권장)
승인 레벨: Level 1 필수
AI 표시: 필수 (법적 책임 명시)
```

### 전화 (자동음성/사람)
```
목적: 긴급 대응 또는 중요한 연락
예시:
  - 야간 수강료 미납 최종 독촉 (사람이 필요하면)
  - 학생 건강 응급 (사람)
  - 법적 통보 (사람 권장)

SLA: 영업시간 1시간 내
개인정보: ✅ 포함
승인 레벨: Level 1 필수
AI 표시: 자동음성은 "[AI 안내]" 필수

규제:
  - 야간(21:00~08:00) 전화 금지 (긴급 제외)
  - 대기시간 상한선: 180초
```

## 응답 시간 SLA (민원 유형별)

| 민원 유형 | 우선순위 | 승인자 | SLA | 채널 |
|---------|---------|-------|-----|------|
| 환불/결제 오류 | P0 (긴급) | 원장 | 1시간 | 앱→카톡 |
| 강사 변경 요청 | P1 (높음) | 원장+실장 | 4시간 | 앱→카톡 |
| 수강료 연체 | P2 (중간) | 시스템 | 24시간 | SMS→카톡 |
| 일반 문의 | P3 (낮음) | 담당자 | 48시간 | 이메일→카톡 |
| 규제/법적 | P0 (긴급) | 원장+변호사 | 2시간 | 앱→이메일 |

## 학부모·수강생 개인정보 포함 메시지 처리 기준

### 수집 단계
```
✅ 필수 수집:
  - 학생 이름, 학년 (교육 목적)
  - 학부모 전화/이메일 (연락용)
  - 수강과목, 월 수강료 (청구용)

⚠️ 조건부 수집:
  - 성적/성향 정보 (학부모 동의 필수)
  - 건강정보 (부상/알레르기 등 - 응급 시만)
  - 가정환경 정보 (맞춤형 상담 시 명시 동의)

❌ 금지:
  - 정치/종교 정보
  - 부모 소득 정보 (공개되는 경우)
```

### 발송 단계
```
기본 원칙: 최소 필요 정보만 포함

수강료 연체 알림:
  ❌ "김민준 학생의 4월 수강료 50만원 미납"
  ✅ "[학생 이름]의 수강료가 미납되었습니다. 확인해주세요."

강사 변경 공지:
  ❌ "김민준은 이전 강사 박수진이 부상으로 퇴사해서..."
  ✅ "[과목] 강사가 변경되었습니다. 4월 15일부터 새 강사와 수업합니다."

성적 관련:
  ❌ "김민준의 이번 달 평균 70점 (반 평균 82점)"
  ✅ "[학생]의 성적 결과가 나왔습니다. 앱에서 확인해주세요."
```

### 동의 관리
```
초기 동의 (가입 시):
  ☐ 교육 목적 정보 수집 (필수)
  ☐ 마케팅 정보 수신 (선택)
  ☐ 강사/담당자가 알림 발송 (필수)

연간 갱신 (1월 또는 계약갱신 시):
  - 동의 현황 재확인
  - 비동의 학부모 리스트 분리 (알림 제외)
```

## AI 생성 메시지 표시 방식

### 투명성 vs 경험의 균형

**원칙**: 정보성 메시지는 투명성 우선, 자동 확인 메시지는 경험 우선

| 메시지 유형 | AI 표시 | 위치 | 예시 |
|-----------|--------|------|------|
| 시스템 안내 | 필수 | 상단 | "[AI 시스템] 수강료가 확인되었습니다" |
| 연체 알림 | 필수 | 상단 | "[자동 알림] 수강료 미납 안내" |
| 원장 판단 결과 | 필수 | 바닥글 | "...원장 승인됨" |
| 법적 통보 | 필수 | 상단+하단 | "AI 생성 메시지 (원장 확인)" |
| 학부모 정보성 | 선택 | 생략 가능 | "수강료 결제가 완료되었습니다" |

## 오류 및 비정상 처리

### AI 메시지 오류 (hallucination 등)
```
학부모가 "메시지가 틀렸어요"라고 이의 제기:

1. 즉시 앱에서 경고 뱃지 표시
2. 원장에게 알림: "[오류 보고] 메시지 검토 필요"
3. 원장이 정정 메시지 승인 후 발송
4. 감사 로그: 원본 + 정정본 + 시간
```

### 부정확한 개인정보 발송
```
학부모: "우리 아이 이름을 잘못 썼어요 (김민준 → 김민수)"

→ 즉시 차단 + 원장 보고 + 정정본 승인 + 감사 기록

기록:
  - 원본 메시지 (아카이브)
  - 발송 시간
  - 정정 시간
  - 영향 범위 (몇 명에게 발송되었는가)
```

## 성공 지표

| 지표 | 목표 | 측정 |
|------|------|------|
| **SLA 준수율** | > 95% | 각 채널별 실제 발송 시간 추적 |
| **오류율** | < 0.5% | 학부모 클레임 수 / 총 발송 메시지 |
| **동의율** | > 80% | 마케팅 동의 학부모 / 전체 |
| **응답율** | > 70% | 앱 알림 확인 학부모 / 발송 대상 |
| **개인정보 위반** | 0건 | 월별 감사 결과 |
