# Paperclip UI 레퍼런스 (개발 참고용)

> HagentOS UI 구현 시 참고할 Paperclip 화면 구조 분석.
> **주의**: 컴포넌트 복제 아님 — 패턴·레이아웃·UX 흐름 참고용.

---

## 레이아웃 4존

```
┌──────┬──────────────────────┬──────────────────────────────────┬──────────────┐
│ Z0   │ Z1                   │ Z2 (Main)                        │ Z3           │
│ 72px │ 240px (w-60)         │ flex-1                           │ 320px        │
│      │ 사이드바              │ 메인 콘텐츠                        │ 속성 패널    │
│ 기관 │ - 기관명 + 검색       │ <BreadcrumbBar sticky>           │ hidden on    │
│ 레일 │ - 새 케이스 버튼      │ <main p-4/p-6>                   │ mobile       │
│      │ - 섹션별 nav          │ <Outlet />                       │              │
└──────┴──────────────────────┴──────────────────────────────────┴──────────────┘
```

**Mobile**: Zone0+Zone1 → 오버레이 슬라이드. Zone3 → 숨김. 하단 탭바 5개 고정.

## 화면별 레이아웃 상세

### 1. 대시보드 (`/[org]/dashboard`)

```
┌─ 에이전트 현황 ──────────────────────────────┐
│ [에이전트명] [상태뱃지]  실행 중: N            │
│ "마지막 실행: 완료 / 실행 중 / 오류"           │
└──────────────────────────────────────────────┘

┌─ 4개 지표 차트 (가로 배열) ──────────────────┐
│ [실행 추이] [케이스 분포] [상태별] [승인률]    │
└──────────────────────────────────────────────┘

┌─ 최근 케이스 ────────────────────────────────┐
│ • 케이스 제목 ················ 날짜 / 상태    │
│ • 케이스 제목                                 │
│ ...                                           │
└──────────────────────────────────────────────┘
```

**차트 4종**: Run Velocity (바), 케이스 우선순위 분포 (스택 바), 케이스 상태 분포, 승인률

### 2. 케이스 목록 (`/[org]/cases`)

```
← Back    필터 ▼   정렬 ▼   그룹 ▼   [표시방식]

─ IN PROGRESS ──────────────────────────────────
  ○ CAS-127  민원 처리: 홍길동 어머니 성적 불만    Feb 26

─ TODO ─────────────────────────────────────────
  ○ CAS-126  이탈 위험: 박민준 (결석 4회)          Feb 26

─ BLOCKED ──────────────────────────────────────
  ⊘ CAS-3   스케줄 충돌: 3월 14일 보강 강사 미배정
  ⊘ CAS-2   법정 기한: 소방 점검 D-7
```

**그룹핑 기준**: 상태 / 유형 / 에이전트 / 우선순위

### 3. 케이스 상세 (`/[org]/cases/:id`) — 3존

## HagentOS → Paperclip 화면 대응표

| HagentOS 화면 | Paperclip 대응 | 차이점 |
|---------------|---------------|--------|
| 대시보드 | Dashboard | 에이전트 현황 + 오늘 현황 카드 추가 |
| 케이스 목록 | Issues | 유형: 민원/이탈/스케줄로 분류 |
| 케이스 상세 | IssueDetail | 에이전트 실행 결과 + 승인 버튼 강조 |
| 승인 큐 | Approvals | HagentOS 핵심 — 카드형 대기 목록 |
| 에이전트 상세 | AgentDetail | 4개 차트 + 실행 이력 |
| 에이전트 조직도 | OrgChart | 계층 카드 트리 |
| k-skill 레지스트리 | Skills | 설치/장착 플로우 추가 |
| 스케줄 캘린더 | (없음) | HagentOS 고유 화면 |
| API 예산 | Costs | 에이전트별 토큰 예산 |
| 처리 이력 | Activity | 감사 로그 타임라인 |
| 자동 실행 | Routines | Heartbeat 스케줄 |
| 알림함 | Inbox | 에이전트 결과 + 실패 알림 |

## 구현 시 핵심 패턴

1. **Live indicator**: 실행 중인 에이전트에 파란 pulse dot + "N live" 카운트
2. **Failure tone**: 알림함 실패 항목 → danger 색상 (빨간 배지)
3. **Collapsible sections**: 사이드바 섹션 (에이전트 팀, 운영 영역) 접힘/펼침
4. **Status grouping**: 케이스 목록의 상태별 그룹 헤더
5. **Properties panel**: 케이스/에이전트 상세에서 오른쪽 320px 패널
6. **Mobile overlay**: 사이드바 → z-50 fixed overlay, 스와이프로 열기/닫기
