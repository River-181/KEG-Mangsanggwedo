# Academy Operations (학원 운영 도메인)

> 한국 학원이 실제로 어떻게 돌아가는지에 대한 도메인 지식 정본.
> 에이전트 설계([[04_ai-agents/]])와 워크플로우 설계([[05_workflows/]])의 기반.

---

## 학원 운영 일과 (Daily Cycle)

학원의 하루는 표면적으로는 수업-출결-수납이지만, 실질적인 운영자 시간은 **예외 처리와 커뮤니케이션 대응**에 소진된다.

| 시간대 | 주요 활동 |
|--------|-----------|
| 오전 (~12시) | 신규 문의 응대, 상담 일정 조율, 전날 미납/미처리 확인, 강사 공결 대타 조정 |
| 오후 (12~17시) | 수업 시작 전 반 배정·보강 확정, 학부모 카카오톡/전화 응대, 수강생 출결 확인 |
| 저녁 (17~21시) | 주요 수업 집중 시간대, 등원 알림 처리, 상담 미팅, 미납 알림 발송 |
| 야간 (21시~) | 수업 종료 후 하원 확인, **심야 카카오톡 민원**(성적 불만, 결석 사유, 강사 이슈) |

> ==핵심 인사이트: 밤 10시 이후에도 카카오톡·문자 기반 비정형 민원이 이어지며, 운영자는 사실상 상시 대기 상태에 놓인다.==

---

## 주간 / 월간 / 연간 사이클

### 주간

- 주간 상담·민원 정리 및 재응대
- 반별 등록률, 재등록률, 결석 추세 점검
- 강사 스케줄·업무량 재조정 (보강, 휴강, 대체수업 확정)
- 교재·프린트·평가 준비 및 결과 공유
- 학부모 공지, 시험/행사/설명회 메시지 발송

### 월간

- 수납 마감, 미수금 추적, 환불·할인·예외 승인
- 급여·프리랜서 정산, 세무 자료 정리
- 사업장현황신고, 현금영수증, 면세 매출 증빙 준비
- 신규 모집 성과 및 재등록·이탈 분석
- 강사 계약, 채용·퇴사, 근로자성 분쟁 위험 점검
- 시설·소방·어린이통학버스·보험·법정교육 일정 점검

> ==데이터 포인트: SaaS 도입 전후 비교 시 수납·성적발송·출결/상담 세 가지가 전체 행정 시간의 60%를 차지. 도입 후 12시간/주 → 2.5시간/주(약 79% 감소) 사례 있음 (온길 OMS 내부 분석).==

### 연간·계절

- **신학기 모집** (2~3월, 8~9월): 상담 폭증, 반 편성 재구성
- **여름·겨울 특강** (7~8월, 12~1월): 별도 시간표·강사 계약·수납 체계
- **수능 대비 시즌** (9~11월): 모의고사 일정 연동, 성적 리포트 빈도 증가
- **법정 점검 주기**: 소방·위생·교육청 공문 대응

## 핵심 업무 영역

### 수강생 관리

**lifecycle**: 신규 문의 → 상담 → 등록 → 반 배정 → 수업 → 재등록 또는 이탈

- **출결**: 등원 확인, 결석·지각 처리, 보강 조정, 학부모 알림
- **상담 기록**: 전화·카카오톡·대면 상담 이력, 담임 메모, 이전 약속 내용
- **성적 관리**: 시험 점수 입력 → 학생별 성적 리포트 생성 → 학부모 발송
- **이탈 방어**: 성적 하락·태도 이슈·결석 누적 시 조기 감지 및 재등록 설득
- **프로필**: 학생 기본정보 + 수강 이력 + 상담 이력 + 결제 이력

### 강사 관리

- **채용·계약**: 근로자성 여부(프리랜서 vs 근로자), 계약서 관리
- **시간표 배정**: 강사별 강의실·과목·시간 조율
- **보강·대체**: 공결 시 대타 강사 섭외, 보강 일정 조율, 학부모 공지
- **성과 평가**: 재등록률·학생 피드백·수업 이탈 연관 분석
- **급여 정산**: 수업 시간 기반 정산, 프리랜서 세금계산서, 원천징수

### 학부모 소통

- **주 채널**: 카카오톡(개인 번호 또는 채널), 문자, 전화
- **주요 내용**: 숙제 안내, 수업 리포트, 성적 공유, 결석 사유 확인, 공지사항
- **민원 유형**: 성적 불만, 강사 태도, 환불 요구, 심야 감정 민원
- **페인 포인트 3종**: (1) 반복 입력, (2) 개인 전화번호 노출, (3) 업무·비업무 카톡이 24시간 혼재

> ==현장 목소리: "수업 끝나고 폰 붙잡고 학부모 카톡 보내는 시간이 매일 1~2시간." 자동화 후 "수업 끝나고 폰 붙잡고 있는 시간이 싹 사라졌다."==

### 상담·입퇴원

- **신규 유입 흐름**: 전화/온라인 문의 → 상담 예약 → 대면/전화 상담 → 등록 클로징 → 반 배정 → 첫 수업
- **상담 클로징**: 원장 또는 상담실장의 판단이 필수인 승인 지점
- **퇴원 프로세스**: 이탈 징후 감지 → 설득 시도 → 퇴원 처리 → 환불 정산
- **환불**: 학원법상 기준 적용, 예외 승인은 원장 직접 처리

### 스케줄링 (7종)

1. 강사 수업 시간표
2. 보강·대체 수업 일정
3. 상담 예약 일정
4. 차량 운행 일정 (해당 학원)
5. 시험·평가 일정
6. 시설·강의실 예약
7. 법정 교육·점검 일정

> ==스케줄 충돌이 가장 빈번한 케이스: 강사 공결 → 보강 일정 재조정 → 강의실 재배정 → 학부모 공지 연쇄 발생.==

### 외부 시험·자격증 일정

학원 유형마다 연동해야 할 외부 시험 일정이 다르다. 이 일정이 학원 전체 운영(특강 편성, 모의고사, 상담 집중 기간)을 좌우한다.

| 학원 유형 | 주요 시험 | 일정 주기 |
|-----------|-----------|-----------|
| 입시/보습 | 수능, 6·9월 모의평가, 학력평가 | 연 1회(수능), 연 2~4회(모의) |
| IT/코딩 | 정보처리기사, SQLD, AWS 자격증, 코딩테스트 | 연 2~4회, 수시 |
| 어학 | TOEIC, TOEFL, IELTS, JLPT, HSK | 월 1~2회, 연 2~4회 |
| 회계/경영 | 전산세무회계, CPA, 감정평가사, 공인중개사 | 연 1~3회 |
| 운전면허 | 필기/기능/도로주행 | 수시 (주 단위) |
| 예체능 | 실기 시험, 대회, 오디션 | 비정기 |

**운영 영향**: 시험 D-30/D-7/D-1 기준으로 특강 편성, 모의고사 실시, 학부모 안내, 상담 집중 기간이 자동으로 트리거됨.

### 행정·운영

- **수납**: 청구서 발행, 미납 추적, 비대면 결제(카카오페이·카드·계좌이체)
- **세무**: 현금영수증 발행, 사업장현황신고, 면세 매출 증빙
- **노무**: 강사 근로자성, 4대보험, 원천징수
- **시설**: 강의실 배정, 에어컨·빔프로젝터 등 장비 관리
- **컴플라이언스**: 소방 점검, 교육청 공문, 어린이통학버스 안전 기준

## 운영자 유형별 역할

| 역할 | 주요 업무 범위 |
|------|---------------|
| **원장** | 최종 승인(환불·채용·예외), 경영 판단, 학부모 에스컬레이션, 세무·노무 책임 |
| **실장/부원장** | 일상 운영 총괄, 강사 스케줄 조율, 민원 1차 대응, 수납 마감 관리 |
| **상담실장** | 신규 문의 응대, 상담 예약·진행, 등록 클로징, 재등록 설득 |
| **강사(담임)** | 수업 진행, 출결 확인, 학생 상태 공유, 학부모 연락, 보강 처리 |
| **행정 직원** | 출결 입력, 수납 처리, 공지 발송, 자료 정리, 전화 응대 |

> 소규모 학원(1~5인)에서는 원장이 상담실장·강사·행정 직원 역할을 동시에 수행한다.

## 학원 유형별 차이

| 유형 | 핵심 pain | 스케줄 특이점 | 민원 패턴 |
|------|-----------|--------------|-----------|
| **보습/입시** | 성적 관리, 재등록 방어, 심야 민원 | 수능·모의고사 연동 필수 | 성적 불만, 강사 교체 요구 |
| **IT/코딩** | 중도 탈락, 취업 기대 관리, 커리큘럼 변경 | 프로젝트 마감 기반 일정 | 취업 성과 미달 민원 |
| **태권도/예체능** | 차량 운행, 안전, 돌봄형 민원 | 차량 시간표가 핵심 | 저연령 학부모 빈번 연락 |
| **어학** | 레벨 테스트, 반 편성 빈도 높음 | 레벨업 시험 주기적 발생 | 반 배정 불만, 강사 국적 이슈 |
| **운전면허** | 차량·시설 고정비, 실습 리스크 | 도로 실습 예약 관리 | 시험 불합격, 일정 변경 |

> 공통 코어: `학생·학부모·강사·케이스·결제·일정·승인·로그`는 유형과 무관하게 존재.
> 유형별 특수 모듈(차량, 취업, 실습)은 domain pack으로 분리 가능.

## 현재 도구 생태계

학원 현장은 단일 솔루션 없이 **여러 도구를 수동으로 연결**해 운영한다.

| 도구 | 용도 | 한계 |
|------|------|------|
| **클래스업 / 온하이** | 출결·수납·공지 | 예외 처리·맥락 관리 불가 |
| **카카오톡 (개인 번호)** | 학부모 소통 | 개인 번호 노출, 업무·비업무 혼재 |
| **카카오 알림톡 (솔라피/알리고)** | 자동 알림 발송 | 비정형 대화 불가 |
| **엑셀 / 구글 시트** | 성적·수납·강사 정산 | 수동 입력, 버전 충돌 |
| **구글 캘린더** | 상담·시험 일정 | 출결·수납 시스템과 미연동 |
| **카카오채널 / 네이버 예약** | 신규 문의 유입 | CRM 연동 없음 |

> ==현재 도구들이 커버하는 영역: 출결·수납·공지 (정형 데이터).
> 커버하지 못하는 영역: 민원 맥락, 케이스 이력, 승인 흐름, 예외 처리 기록.==

## 예외 vs 루틴 비율

> ==핵심 인사이트: 운영자 시간의 상당 부분이 예외 처리에 소진된다.==

**루틴 업무** (도구가 커버 가능):
- 출결 자동 기록 및 등하원 알림
- 정기 수납 청구서 발행
- 반복 공지 발송

**예외 처리 업무** (사람이 직접 처리):
- 강사 공결 → 보강·대타 조율 → 학부모 재공지
- 성적 불만·환불 요구·심야 감정 민원
- 재등록 설득 (정서적 판단 필요)
- 반 편성 이슈, 강사 교체 요구
- 차량 사고·시설 고장·안전 이슈
- 교육청 점검, 세무·노무 문의 대응

**예외 처리를 어렵게 만드는 구조적 원인**:
1. **맥락 분산**: 이전 상담 내용이 카톡·전화·노트에 흩어짐
2. **약속 추적 불가**: 누가 무엇을 약속했는지 기록 없음
3. **누적 여부 불명**: 이번 민원이 처음인지 반복인지 모름
4. **에스컬레이션 수동**: 강사→실장→원장 보고가 비정형

## HagentOS 매핑 힌트

각 업무 영역이 어떤 에이전트/워크플로우와 연결되는지 간단한 참조 테이블.
상세 설계는 [[04_ai-agents/]] 및 [[05_workflows/]] 참조.

| 도메인 영역 | 연결 에이전트 (예상) | 핵심 데이터 |
|-------------|---------------------|-------------|
| 학부모 민원 응대 | Complaint Agent | 학생 프로필, 상담 이력, 강사 맥락 |
| 출결·보강 조율 | Scheduler Agent | 강사 시간표, 강의실, 학부모 연락처 |
| 재등록/이탈 방어 | Retention Agent | 출결 추세, 성적 이력, 상담 기록 |
| 신규 상담·등록 | Intake Agent | 문의 이력, 반 현황, 수납 설정 |
| 수납·청구 | Finance Agent | 수강생 목록, 수납 이력, 결제 수단 |
| 성적 리포트 발송 | Analytics Agent | 시험 점수, 성적 분석 템플릿 |
| 강사 스케줄·정산 | Staff Agent | 강사 계약, 수업 이력, 정산 기준 |
| 법무·컴플라이언스 | Compliance Agent | 학원법, 소방, 노무 체크리스트 |

> 사람이 반드시 남아야 하는 승인 지점: 신규 등록 클로징, 환불·할인 예외 승인, 강사 채용·해고, 법적·악성 민원 대응, 이탈 방어 정서적 설득.
