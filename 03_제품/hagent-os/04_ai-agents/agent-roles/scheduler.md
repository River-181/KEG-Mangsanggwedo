# Scheduler Agent (스케줄 관리 담당)

> 학원의 7종 스케줄을 단일 뷰로 통합하고, 충돌을 자동 감지해 해결안을 제안한다.
> 다른 에이전트가 스케줄 맥락을 참조하는 **운영 OS의 핵심 레이어**.

---

## 역할 요약

| 항목 | 내용 |
|------|------|
| **목적** | 7종 스케줄 통합 관리 · 충돌 감지 · 외부 시험 일정 연동 · 기한 알림 |
| **소유자** | 실장 / 원장 |
| **트리거** | cron (매일 오전 7시) / assignment (스케줄 변경 이벤트 발생 시 즉시) |
| **출력물** | 오늘의 스케줄 브리핑 카드 / 충돌 경고 카드 / D-7·D-1 기한 알림 |
| **제로휴먼 레벨** | Level 0 일상 업데이트 · Level 1 충돌 해결 제안 (실행은 승인 후) |

## 7종 스케줄 관리

| # | 유형 | 핵심 데이터 | 충돌 케이스 |
|---|------|------------|------------|
| 1 | **강사 수업 시간표** | 강사 ID · 강의실 · 요일·시간 · 과목 | 강의실 중복, 강사 이중 배정 |
| 2 | **보강·대체 수업** | 원강사 · 대체강사 · 보강일시 · 해당 반 | 대체 강사 공가용 여부, 강의실 가용 여부 |
| 3 | **상담 예약** | 상담 유형 · 학부모·학생 · 담당자 · 상담실 | 상담실 중복, 담당자 스케줄 충돌 |
| 4 | **차량 운행** | 루트 · 출발시간 · 탑승 학생 목록 · 기사 | 학생 추가·변경 시 좌석 초과 |
| 5 | **시험·평가** | 반 · 시험 날짜 · 범위 · 장소 | 같은 강의실 시험 중복 |
| 6 | **시설 예약** | 강의실·체육관·상담실 · 예약자 · 시간대 | 시설 중복 예약 |
| 7 | **법정 기한·점검** | 항목명 · 마감일 · 담당자 · 법령 근거 | D-7·D-1 알림 미발송 |

## 외부 시험 일정 연동

학원 유형별로 연동할 시험 일정이 다르다. 이 일정은 특강 편성·상담 집중 기간을 자동 트리거한다.

| 학원 유형 | 연동 시험 | 자동 트리거 이벤트 |
|-----------|-----------|-----------------|
| 입시·보습 | 수능, 6·9월 모의평가, 학력평가 | D-30 특강 편성 제안, D-7 학부모 안내 초안 |
| IT·코딩 | 정보처리기사, SQLD, AWS, 코딩테스트 | D-14 집중반 편성 제안, D-1 응시 체크리스트 |
| 어학 | TOEIC(월 2회), JLPT(연 4회), HSK | 접수 마감 D-7 알림, 시험 D-1 준비 안내 |
| 회계·경영 | 전산세무회계, 공인중개사, CPA | D-30 특강 편성, D-7 모의고사 일정 삽입 |

> 시험 일정 데이터 소스: `exam-schedule-sync` k-skill → 공공데이터 API + 자격증 시험 기관 공개 캘린더.

## 실행 트리거 상세

```
[Trigger: cron — 매일 오전 7시]
  1. 당일 스케줄 전체 로드 (7종)
  2. 충돌 감지 실행
  3. D-7·D-1 법정 기한 체크
  4. 외부 시험 D-day 계산
  5. 오늘의 브리핑 카드 생성 → 승인 대시보드 Push
  6. 구글 캘린더 동기화 상태 확인

[Trigger: assignment — 스케줄 변경 이벤트]
  예: 강사 공결 접수, 상담 예약 요청, 시설 예약 등록
  1. 변경 내용 수신
  2. 충돌 즉시 감지
  3. 해결안 생성 → 담당자 알림 카드 Push
  4. 구글 캘린더 단방향 업데이트
```

## 스케줄 충돌 감지 및 해결 패턴

충돌 유형별 자동 해결안 생성 로직:

### 강의실 중복
- 감지: 동일 강의실 · 동일 시간대에 2개 이상 예약
- 해결안: 가용 강의실 목록 조회 → 대안 2개 제시 → 담당자 선택 요청

### 강사 이중 배정
- 감지: 동일 강사 · 겹치는 시간대에 복수 수업
- 해결안: 강사 가용 시간 재조회 → "해당 수업을 다른 강사로 대체" OR "시간 이동" 옵션 제시

### 상담 시간 충돌
- 감지: 상담 담당자 일정 중복
- 해결안: 빈 시간 슬롯 3개 제안 → Retention/Intake Agent에 전달해 재예약 요청

## 강사 공결 → 대체 강사 자동 매칭 흐름

```
[강사 공결 접수]
    ↓
Scheduler Agent: 공결 강사의 당일 수업 목록 추출
    ↓
substitute-matcher k-skill: 
  - 동일 과목 가능 강사 필터
  - 해당 시간 가용 강사 필터
  - workload 초과 여부 체크
    ↓
대체 강사 후보 1~3인 제안 카드 생성
    ↓
[원장/실장 승인 — 원클릭 선택]
    ↓
Scheduler DB 업데이트 + 구글 캘린더 반영
    ↓
Notification Agent: 해당 반 학부모 보강 공지 발송
```

> 대체 강사 후보가 없을 경우: "가용 강사 없음 — 보강 일정 재조정 필요" 카드로 에스컬레이션.

## 구글 캘린더 동기화

| 방향 | 내용 | 주기 | 상태 |
|------|------|------|------|
| HagentOS → 캘린더 | DB 스케줄 변경 즉시 반영 | 실시간 (webhook) | **MVP ✅** |
| 캘린더 → HagentOS | 원장이 캘린더에서 직접 수정한 항목 감지 | cron 10분 간격 | v1.1 이후 |

**MVP 구현 (v1.0):**
- 단방향 동기화만 지원: HagentOS → Google Calendar
- OAuth 도메인 검증 완료 후 양방향 활성화

**향후 (v1.1):**
- 동기화 충돌(같은 이벤트 양쪽 동시 수정) 시: HagentOS 버전을 정본으로 간주, 충돌 내역을 감사 로그에 기록 후 원장에게 알림.
- 구글 캘린더 MCP 사용: `google-calendar-sync` k-skill 경유.

## 법정 기한 D-7/D-1 알림 트리거

| 기한 유형 | D-7 액션 | D-1 액션 |
|-----------|---------|---------|
| 사업장현황신고 | 브리핑 카드 + 담당자 알림 | 긴급 알림 Push |
| 소방 점검 | 체크리스트 초안 생성 | 당일 알림 |
| 어린이통학버스 정기 점검 | 점검 예약 확인 요청 | 긴급 알림 |

| 현금영수증 발급 기한 | Finance Agent에 처리 요청 | 긴급 알림 |

## 제로휴먼 레벨 정의

| Level | 설명 | Scheduler Agent 행동 |
|-------|------|---------------------|
| **Level 0** | 자동 업데이트 | 일상 스케줄 변경(반복 수업 등록, 캘린더 동기화) 자동 처리 |
| **Level 1** | 충돌 해결 제안 | 충돌 감지 + 해결 옵션 제시 → 담당자 선택 대기 |
| Level 2 | 승인 대기 | 강사 대체, 시설 재배정 등 변경 사항 원장 확인 |
| Level 3 | 승인 후 실행 | 학부모 보강 공지 발송, 외부 캘린더 대규모 변경은 승인 필수 |

## System Prompt 초안

```
당신은 HagentOS의 Scheduler Agent입니다.
한국 학원의 7종 스케줄을 통합 관리하고, 충돌을 감지해 실행 가능한 해결안을 제시합니다.

[입력]
- schedules: 오늘 기준 전체 스케줄 목록 (7종)
- events: 당일 변경 이벤트 목록 (공결, 예약, 취소 등)
- deadlines: 법정 기한 목록 (마감일, 유형, 담당자)
- exam_calendar: 외부 시험 일정 (학원 유형에 맞게 필터됨)
- today: 오늘 날짜

[처리 원칙]
1. 충돌을 빠짐없이 감지하라. 누락된 충돌은 현장 혼선 직결이다.
2. 해결안은 반드시 실행 가능한 옵션으로 제시하라. "검토 필요"만 있는 카드는 금지.
3. D-7·D-1 기한은 매일 반드시 체크하라. 놓치면 법적·행정적 리스크가 발생한다.
4. 대체 강사 제안 시 workload 과부하(주 20시간 초과) 강사는 제외하라.
5. 구글 캘린더 동기화 오류 발생 시 감사 로그에 기록하고 원장에게 즉시 알림.
6. 학부모·학생 개인정보(연락처 등)는 출력 카드에 포함하지 마라.

[출력 형식 — JSON]
{
  "run_date": "YYYY-MM-DD",
  "daily_brief": { "total_schedules": 0, "changes_today": 0 },
  "conflicts": [ { "type", "description", "options": [] } ],
  "deadline_alerts": [ { "item", "days_remaining", "urgency" } ],
  "exam_alerts": [ { "exam_name", "date", "days_remaining", "suggested_action" } ],
  "sync_status": { "google_calendar": "ok | error", "last_synced": "ISO8601" }
}
```

## 다른 에이전트와의 협력

| 요청 에이전트 | 협력 내용 | 흐름 |
|-------------|---------|------|
| **Complaint** | 민원 처리 시 강사 스케줄 참조 | Complaint가 "강사 A 오늘 수업 있나?" 질의 → Scheduler 응답 |
| **Retention** | 이탈 위험 학생 상담 예약 | Retention이 상담 요청 → Scheduler가 빈 슬롯 반환 → 예약 생성 |
| **Intake** | 신규 등록 상담 일정 잡기 | Intake가 가용 상담 시간 요청 → Scheduler 슬롯 제공 |
| **Staff** | 강사 공결 대체 매칭 | Staff가 공결 접수 → Scheduler가 대체 강사 매칭 흐름 실행 |
| **Compliance** | 법정 기한 D-day 알림 | Scheduler가 D-7·D-1 감지 → Compliance Agent에 체크리스트 생성 요청 |
| **Notification** | 보강·변경 학부모 공지 | Scheduler 변경 확정 후 → Notification Agent에 발송 지시 |
