# Complaint Agent (민원 담당)

> 학부모·수강생 민원을 분류하고 응답 초안을 생성하는 에이전트.

---

## 역할 요약

민원 담당 에이전트는 학원 원장의 ==가장 큰 시간 도둑==인 비정형 민원을 자동으로 처리한다.
접수된 민원을 분류·긴급도 판정하고, 학원 문화에 맞는 응답 초안을 생성해 원장 승인 대기 상태로 전달한다.
야간·주말 접수 민원도 실시간으로 처리하되, **최종 발송은 항상 원장 승인을 거친다.**

| 속성 | 값 |
|------|-----|
| agentId | `complaint` |
| role | `specialist` |
| 제로휴먼 레벨 | Level 1 (초안 생성 + 원클릭 승인) |
| 복잡 건 | Level 2 (에스컬레이션 → 원장 직접 처리) |

---

## 민원 유형 분류 체계 (7종)

[[communication-model]] 기반. 에이전트는 접수 즉시 아래 분류 중 하나로 태깅한다.

| 코드 | 분류 | 예시 | 기본 심각도 | 기본 대응 주체 |
|------|------|------|------------|--------------|
| `complaint.grade` | 성적 불만 | "점수가 왜 이렇게 낮아요", "수업 효과가 없어요" | 중~고 | 강사 + 원장 |
| `complaint.behavior` | 행동 지적 | "강사 말투가 문제예요", "학생 태도 지적" | 중 | 원장 |
| `complaint.refund` | 환불 요청 | "퇴원할게요 환불 해주세요", "과납 환급" | 고 | 원장 |
| `complaint.schedule` | 스케줄 조정 | "시간대 바꿔주세요", "반 이동 요청" | 저~중 | 실장 + 원장 |
| `complaint.teacher` | 강사 교체 요구 | "선생님 바꿔주세요", "수업 방식 맞지 않아요" | 고 | 원장 |
| `complaint.facility` | 시설·안전 불만 | "냉방이 안 돼요", "차량 지연 반복" | 중 | 원장 + 시설 |
| `complaint.info` | 공지 오류·혼선 | "일정이 다르게 왔어요", "잘못된 안내" | 저 | 실장 |

## 처리 흐름

```
1. 접수
   카카오봇 / 대시보드 직접 입력 / 오케스트레이터 지시
       ↓
2. 분류
   민원 유형(7종) + 긴급도(즉시/당일/주간) 자동 태깅
   + 관련 학생·학부모 컨텍스트 조회 (기존 민원 이력, 출결, 납부 상태)
       ↓
3. 초안 생성
   분류 결과 기반 응답 초안 작성
   - 공감 표현 → 사실 확인 요청 또는 조치 약속 → 다음 단계 안내
   - 에스컬레이션 여부 판단 (조건 참조)
       ↓
4. 승인 대기 (pending_approval)
   대시보드에 카드 표시: [민원 유형] [긴급도] [초안 미리보기]
   원장: 승인 / 편집 후 발송 / 반려
       ↓
5. 발송
   승인 시 카카오봇 or 이메일로 학부모에게 자동 발송
       ↓
6. 기록
   complaint_log에 케이스 저장
   (접수시각, 유형, 긴급도, 초안, 수정 내용, 발송 시각, 결과)
```

## 제로휴먼 레벨

| 레벨 | 적용 조건 | 에이전트 행동 |
|------|----------|-------------|
| **Level 0** | 즉각 자동 처리 (긴급 알림 등) | 사람 개입 없이 자동 발송 (사전 템플릿 승인 기반) |
| **Level 1** | 초안 + 원클릭 승인 (정형 민원) | 초안 생성 → 원클릭 승인 → 자동 발송 |
| **Level 2** | 초안 + 편집 후 발송 (복잡·고심각도 민원) | 초안 + 에스컬레이션 플래그 → 원장 직접 수정 유도 |

---

## 긴급도 판단 기준

에이전트는 민원 텍스트 분석 + 컨텍스트를 종합해 긴급도를 결정한다.

| 긴급도 | 기준 | 응답 목표 시간 | 알림 채널 |
|--------|------|--------------|---------|
| **즉시** | 환불 요청 + 강한 감정 표현 / 강사 교체 요구 + "바로" / 안전 관련 | 2시간 이내 | 푸시 알림 즉시 발송 |
| **당일** | 성적 불만 / 강사 교체 요구 (일반) / 행동 지적 | 당일 업무시간 내 | 대시보드 알림 |
| **주간** | 스케줄 조정 / 공지 오류 / 시설 불만 (비긴급) | 3영업일 이내 | 일일 브리핑에 포함 |

**긴급도 상향 트리거**: 동일 학부모의 3번째 이상 민원 → 자동으로 한 단계 상향.

| 스킬 | 용도 |
|------|------|
| `complaint-classifier` | 민원 유형·긴급도 자동 분류 (fine-tuned on 한국 학원 민원 패턴) |

| `parent-sentiment-tracker` | 민원 감정 강도 측정 (냉정/불만/분노/위기) → 긴급도 보정 |

## System Prompt 초안

```
당신은 HagentOS의 민원 담당 에이전트입니다.
학원 학부모·수강생의 민원을 처리하고 원장을 위한 응답 초안을 작성합니다.

[기관 정보]
- 기관명: {orgName}
- 운영 철학/톤: {orgTone}  (예: 친근, 프리미엄, 성과중심)
- 오늘 날짜·시간: {datetime}

[민원 정보]
- 접수 채널: {channel}
- 민원 원문: {complaintText}

[학부모·학생 컨텍스트]
- 학생명: {studentName} / 등록 기간: {enrolledSince}
- 기존 민원 횟수: {previousComplaintCount}건
- 최근 출결: {attendanceSummary}
- 납부 상태: {paymentStatus}

---

다음 JSON 형식으로 응답하세요:

{
  "type": "complaint.grade|behavior|refund|schedule|teacher|facility|info",
  "urgency": "immediate|today|weekly",
  "escalate": true|false,
  "escalateReason": "에스컬레이션 사유 (escalate=true일 때만)",
  "summary": "민원 한 줄 요약 (원장 대시보드 표시용, 30자 이내)",
  "draft": "학부모에게 발송할 응답 초안 (아래 가이드 준수)",
  "internalNote": "원장에게만 보이는 처리 메모 (배경·권고 조치 등)"
}

[응답 초안 작성 가이드]
1. 첫 문장: 민원을 접수했음을 확인하고 불편에 공감 표현
   (예: "말씀해 주셔서 감사합니다. 불편을 드려 정말 죄송합니다.")
2. 중간: 사실 확인 중임을 알리거나, 조치 약속을 구체적으로 명시
   (예: "해당 사항을 담당 강사와 함께 확인하고 {날짜}까지 연락드리겠습니다.")
3. 마지막 문장: 추가 문의 환영 + 서명
4. 톤: {orgTone}에 맞게 조정. 기본은 정중하고 따뜻하게.
5. 길이: 5문장 이내. 카카오톡 발송 기준.
6. 환불·강사 교체 등 의사결정이 필요한 건: 구체적 약속 대신 "확인 후 연락" 표현 사용.
7. 감정적 민원일수록 공감 표현을 먼저, 사실 확인은 두 번째.
```

## 에스컬레이션 조건

아래 조건 중 하나라도 해당하면 `escalate: true`를 반환하고 Compliance Agent에 알림.

| 조건 | 이유 |
|------|------|
| `complaint.refund` + 금액 언급 또는 법적 언급 | 환불 규정·법적 의무 검토 필요 |
| `complaint.teacher` + "신고", "교육청" 단어 포함 | 민원 공식화 가능성, 법적 리스크 |
| `complaint.behavior` + 아동 안전·폭력 키워드 | 즉각 보고 의무 대상 가능성 |
| 동일 학부모 민원 5회 이상 (30일 이내) | 반복 민원 → 관계 관리 전환 필요 |
| 긴급도 = 즉시 + escalate 판단 신뢰도 높음 | 원장 직접 개입이 더 효과적 |

에스컬레이션 시 Compliance Agent에 전달하는 정보: 민원 원문, 분류, 학부모 이력, 초안(참고용).

## 성공 지표 (KPI)

| 지표 | 목표 | 측정 방법 |
|------|------|---------|
| 민원 초안 생성 소요 시간 | 접수 후 3분 이내 | complaint_log.created_at vs draft_at |
| 원장 초안 수용률 (무편집 승인) | 60% 이상 | approved_without_edit / total_approved |
| 민원 응답 발송까지 걸린 시간 | 긴급: 2시간 / 당일: 8시간 | draft_at vs sent_at |
| 에스컬레이션 정확도 | false positive < 20% | 원장이 에스컬레이션 취소한 비율 |
| 민원 재발률 (동일 학부모 30일 내 재접수) | 15% 이하 | complaint_log 집계 |
