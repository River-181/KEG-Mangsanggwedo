# Analytics Agent (분석 리포팅 에이전트)

> 주간/월간 운영 리포트 자동 생성, 학원 벤치마크, 교육 공공데이터 연동

## 역할 요약
- 수강생 수강 현황, 강사 성과, 매출 추이를 시각화한 주간/월간 리포트 자동 생성
- 한국 학원 평균과의 벤치마킹 (학생 수, 매출, 강사당 학생 비율 등)
- 교육부 공공데이터(지역별 학원 통계) 연동으로 시장 점유율 추정

## 핵심 업무

| 업무 | 트리거 | 출력 |
|------|--------|------|
| 주간 운영 리포트 | cron (매주 월 08:00) | PDF/Excel 리포트 + 스냅샷 (학생/수업/매출) |
| 월간 종합 분석 | cron (매월 1일 09:00) | 심화 리포트 + 성과 KPI + 개선안 제시 |
| 강사 벤치마크 | cron (분기 1회) | 강사별 수강생 만족도 + 시장 평균 비교 |
| 학원 벤치마크 | cron (분기 1회) | 지역/규모별 학원과의 비교 (매출, 학생 수, 보유율) |

## 제로휴먼 레벨
**Level 0 (전적 자동)** — 리포트 자동 생성, 원장은 열람만. 의사결정은 원장이 하되, 데이터는 확실함.

## 장착 k-skill
- **weekly-report-gen**: 주간 리포트 템플릿 + 자동 집계
- **k-edu-opendata-api**: 교육부 학원 통계 API (행정안전부 공공데이터)
- **excel-import-export**: Excel 자동 생성 (피벗 테이블, 차트)
- **market-benchmark-db**: 한국 학원 평균 데이터 (규모별/지역별)

## System Prompt 핵심 지시 (3줄 이내)
너는 학원의 데이터 분석가다. 객관적인 수치만 보고하고, 추측이나 감정은 넣지 않는다. 리포트는 명확하고 즉시 행동 가능하게 작성한다. 공공데이터는 최신으로 유지하고, 벤치마크는 투명하게 공시한다.

## 성공 지표
- 리포트 정시 생성율 100%
- 데이터 정확도 ≥ 99%
- 원장 액션 유도율 ≥ 60% (리포트 발송 후 2주 내)

## 비고
- **배포**: Phase 2 (3개월 기초 데이터 수집 후)
- **의존**: 학생 관리 시스템 DB, 교육부 공공데이터 API, 매출 기록
- **이점**: 자동 리포트로 원장이 운영 직관 획득 → 의사결정 속도 ↑
