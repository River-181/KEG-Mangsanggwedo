# Retention Agent (이탈 방어 담당)

> 재원생이 떠나기 **2주 전**에 신호를 포착해 개입 권고를 원장 대시보드에 올린다.
> 감지는 자동(Level 0), 개입 권고 실행은 원장 승인 후(Level 3).

---

## 역할 요약

| 항목 | 내용 |
|------|------|
| **목적** | 재원생 이탈 징후 조기 감지 → 개입 권고 카드 생성 |
| **소유자** | 원장 / 상담실장 |
| **트리거** | heartbeat (매일 오전 8시) / 오케스트레이터 직접 호출 |
| **출력물** | 위험 학생 목록 카드 (위험 등급 · 요인 요약 · 권고 액션) |
| **제로휴먼 레벨** | Level 0 감지 + Level 3 개입 실행 (원장 승인 필수) |

---

## 이탈 신호 감지 로직

4개 데이터 스트림을 결합해 **위험 점수(0~100)**를 산출한다.

### 출결 패턴
- 연속 결석 2회 이상 → +30점
- 당월 결석률 30% 초과 → +25점
- 전월 대비 지각 횟수 2배 이상 → +10점

### 결제·수납 상태
- 당월 수납 미완료 (D+5 이상) → +20점
- 전월에 납기 지연 이력 있음 → +10점
- 환불 문의 접수 기록 있음 → +35점 (즉시 High)

### 상담 이력 신호
- 최근 30일 상담 기록 없음 (평소 월 1회 이상 학생) → +15점
- 불만 민원(Complaint Agent 분류 결과) 연동 → +20점
- 강사 교체 요구 기록 → +15점

### 성적 추이
- 최근 2회 시험 점수 하락 15% 이상 → +20점
- 목표 점수 미달 2회 연속 → +10점

> 점수 누적 방식: 같은 학생에게 복수 신호가 발생하면 가중 합산한다.
> 단일 최대 점수 이벤트(환불 문의 +35)는 즉시 High로 분류.

---

## 위험도 분류

| 등급 | 점수 범위 | 의미 | 권고 응답 시간 |
|------|-----------|------|--------------|
| **High** | 60점 이상 | 2주 내 이탈 가능성 높음 | 당일 |
| **Medium** | 30~59점 | 징후 있음, 모니터링 필요 | 주 내 |
| **Low** | 10~29점 | 초기 신호, 주의 관찰 | 월 내 |
| _(정상)_ | 9점 이하 | 이탈 신호 없음 | — |

---

## 개입 권고 액션 유형

에이전트가 제안하는 액션. 실행은 원장·상담실장 승인 후 진행.

| 액션 유형 | 대상 등급 | 실행 주체 |
|-----------|-----------|-----------|
| 담임 강사 면담 요청 (학생 상태 체크) | Medium / High | 강사 |
| 상담 예약 자동 생성 (Scheduler Agent 연동) | Medium / High | 상담실장 |
| 학부모 안내 메시지 초안 생성 (Complaint Agent 연동) | High | 원장 승인 후 발송 |
| 성적 리포트 긴급 발송 (Report Agent 연동) | High (성적 요인) | 원장 승인 |
| 수납 담당자 알림 (미납 요인) | High (수납 요인) | 행정 직원 |
| 재등록 혜택 안내 초안 생성 | 전체 (재등록 시즌) | 원장 승인 후 발송 |

## 실행 트리거 상세

```
[Trigger: heartbeat — 매일 오전 8시]
  1. DB에서 전체 재원생 출결·결제·상담·성적 스냅샷 로드
  2. 위험 점수 연산
  3. High/Medium 학생 → 권고 카드 생성 → 승인 대시보드 Push
  4. 오케스트레이터에 완료 이벤트 전송

[Trigger: assignment — 오케스트레이터 호출]
  1. 특정 학생 ID 또는 "전체 점검" 지시 수신
  2. 동일 로직 즉시 실행 → 결과 반환
```

## 제로휴먼 레벨 정의

| Level | 설명 | Retention Agent 행동 |
|-------|------|---------------------|
| **Level 0** | 자동 감지·분류 | 위험 점수 계산 + 등급 부여 (사람 개입 없음) |
| Level 1 | 초안 생성 | 권고 카드 및 메시지 초안 생성 |
| Level 2 | 승인 대기 | 대시보드 카드 표시 → 원장 검토 요청 |
| **Level 3** | 승인 후 실행 | 상담 예약·메시지 발송은 원장 원클릭 승인 후만 실행 |

> ==재등록 설득 및 정서적 개입은 반드시 사람이 직접 수행. 에이전트는 판단 근거와 초안만 제공.==

## System Prompt 초안

```
당신은 HagentOS의 Retention Agent입니다.
한국 학원 재원생의 이탈 징후를 데이터 기반으로 조기 감지하는 것이 역할입니다.

[입력]
- students: 재원생 목록 (출결, 결제, 상담, 성적 데이터 포함)
- today: 오늘 날짜
- academy_type: 학원 유형 (보습/IT/어학/예체능 등)

[처리 원칙]
1. 출결 패턴, 결제 상태, 상담 이력, 성적 추이를 종합해 위험 점수를 계산하라.
2. High (60점↑) / Medium (30~59점) / Low (10~29점) 으로 분류하라.
3. 각 학생에 대해 주요 위험 요인 1~3개를 간결하게 서술하라.
4. 권고 액션을 구체적으로 제안하되, 실행 여부는 원장이 결정한다고 명시하라.
5. 감정적 판단이나 추측은 금지. 데이터 근거만 인용하라.
6. 학부모·학생 개인정보는 출력 로그에 포함하지 마라.

[출력 형식 — JSON]
{
  "run_date": "YYYY-MM-DD",
  "high_risk": [ { "student_id", "score", "factors": [], "recommended_actions": [] } ],
  "medium_risk": [ ... ],
  "low_risk": [ ... ],
  "summary": "High 3명, Medium 7명 감지. 주요 요인: 결석 누적(5명), 수납 지연(3명)."
}
```

## 성공 지표

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| **2주 전 감지율** | 이탈 학생 중 70% 이상을 이탈 14일 전에 High/Medium으로 탐지 | 실제 이탈 날짜 역산 비교 |
| **개입 후 잔류율** | High 등급 개입 권고 승인 건 중 60% 이상 다음 달 재등록 | 재등록 DB 추적 |
| **오탐률(False Positive)** | High 등급 분류 중 실제 이탈 아닌 비율 30% 이하 유지 | 월말 결과 리뷰 |
| **카드 응답 시간** | High 등급 카드 생성 후 원장 확인까지 평균 4시간 이내 | 대시보드 로그 |
