# Workflow: 스케줄 관리 & 강사 공결 대응

> Scheduler Agent가 주도하는 7종 스케줄 통합 운영 워크플로우.
> 일상 운영(Daily Cycle)부터 공결 대응, 외부 시험 연동, 법정 기한 알림까지 커버한다.

---

## 정상 운영 (Daily Cycle)

매일 오전 7시 cron 트리거로 다음 루프가 자동 실행된다.

```
[07:00 cron]
  1. 7종 스케줄 전체 로드
     - 강사 수업 시간표 / 보강·대체 / 상담 예약 / 차량 운행
     - 시험·평가 / 시설 예약 / 법정 기한·점검
  2. 충돌 감지 실행 (schedule-optimizer k-skill)
  3. D-7·D-1 법정 기한 체크
  4. 외부 시험 D-day 계산 (exam-schedule-sync k-skill)
  5. 오늘의 브리핑 카드 생성 → 원장/실장 승인 대시보드 Push
  6. 구글 캘린더 동기화 상태 확인
```

**브리핑 카드 포함 항목**
- 당일 수업 건수, 상담 건수, 차량 운행 여부
- 충돌 감지 결과 (없으면 "정상")
- D-day 기한 알림 (해당일만 표시)
- 어제 대비 변경 스케줄 요약

## 강사 공결 → 대체 매칭 흐름

공결은 학원에서 가장 빈번한 연쇄 예외다. 이 흐름은 assignment 트리거로 즉시 실행된다.

```
[공결 접수] ← Staff Agent 또는 강사 직접 앱 입력
    │
    ▼
Scheduler Agent: 공결 강사의 당일·금주 수업 목록 추출
    │
    ▼
substitute-matcher k-skill 실행:
  - 동일 과목 가능 강사 풀 조회
  - 해당 시간대 가용 여부 필터
  - 주간 workload 초과(20시간↑) 강사 제외
    │
    ├── 후보 있음 → 대체 강사 후보 1~3인 카드 생성
    │                원장/실장에게 원클릭 승인 요청
    │                    │
    │                    ▼ [승인]
    │              Scheduler DB 업데이트
    │              구글 캘린더 반영 (google-calendar-sync)
    │              Notification Agent: 해당 반 학부모 보강 공지 발송
    │
    └── 후보 없음 → "가용 강사 없음 — 보강 일정 재조정 필요" 카드
                    원장에게 에스컬레이션 (보강 날짜 확정 대기)
```

**보강 공지 발송 내용 (Notification Agent)**
- 대상: 해당 반 학부모 전원
- 채널: 카카오 알림톡 (aligo-sms-mcp 백업)
- 내용: 공결 강사명, 대체 강사명, 수업 날짜·시간, 보강 여부

## 구글 캘린더 동기화 흐름

| 방향 | 트리거 | 처리 방식 |
|------|--------|-----------|
| HagentOS → 캘린더 | DB 변경 즉시 | webhook (실시간) |
| 캘린더 → HagentOS | 원장 직접 수정 감지 | cron 10분 간격 |

**동기화 충돌 처리**
- 같은 이벤트 양쪽 동시 수정 시: HagentOS 버전을 정본으로 유지
- 충돌 내역 감사 로그 자동 기록
- 원장에게 "충돌 감지 — HagentOS 버전이 유지되었습니다" 알림 발송

**동기화 오류 시**
- 오류 내용을 감사 로그에 기록
- 원장에게 즉시 알림 (재시도는 다음 cron 사이클에서 자동 처리)

## 상담 일정 자동 생성 (Retention/Intake Agent 요청 시)

Retention Agent 또는 Intake Agent가 상담 슬롯을 요청할 때의 흐름.

```
[요청 에이전트] "상담 슬롯 필요 — 학부모 A, 가능한 날짜 3개"
    │
    ▼
Scheduler Agent:
  - 상담실 가용 시간대 조회
  - 담당 상담사 일정 확인 (충돌 제외)
  - 가용 슬롯 3개 추출 → 요청 에이전트에 반환
    │
    ▼
[요청 에이전트] 학부모에게 슬롯 제안 → 선택 대기
    │
    ▼ [학부모 선택]
Scheduler Agent: 상담 예약 DB 등록 + 캘린더 반영
Notification Agent: 학부모 예약 확인 문자 발송 (aligo-sms-mcp)
```

## 외부 시험 일정 연동 (D-30/D-7/D-1 자동 트리거)

`exam-schedule-sync` k-skill이 학원 유형별 시험 일정을 수집하고, Scheduler Agent가 D-day 계산 후 액션을 트리거한다.

| D-day | 트리거 액션 |
|-------|-------------|
| **D-30** | 특강 편성 제안 카드 생성 → 원장 승인 요청 |
| **D-7** | 학부모 안내 초안 생성 → Notification Agent 대기 상태로 준비 |
| **D-1** | 응시 체크리스트 발송 → 해당 반 학부모 즉시 발송 |

**학원 유형별 연동 시험 예시**

| 학원 유형 | 주요 시험 |
|-----------|-----------|
| 입시·보습 | 수능, 6·9월 모의평가, 학력평가 |
| IT·코딩 | 정보처리기사, SQLD, AWS, 코딩테스트 |
| 어학 | TOEIC (월 2회), JLPT (연 4회), HSK |
| 회계·경영 | 전산세무회계, 공인중개사, CPA |

> 시험 일정 미수집 시: "시험 일정 동기화 실패 — 수동 확인 필요" 카드 생성.

## 스케줄 충돌 감지 & 해결

`schedule-optimizer` k-skill이 충돌을 감지하고 Scheduler Agent가 해결안을 제시한다.

### 강의실 중복
- **감지**: 동일 강의실 · 동일 시간대 2개 이상 예약
- **해결안**: 가용 강의실 목록 조회 → 대안 2개 제시 → 담당자 선택 요청

### 강사 이중 배정
- **감지**: 동일 강사 · 겹치는 시간대 복수 수업
- **해결안**: "다른 강사로 대체" 또는 "시간 이동" 옵션 2개 제시

### 상담 시간 충돌
- **감지**: 상담 담당자 일정 중복
- **해결안**: 빈 시간 슬롯 3개 제안 → Retention/Intake Agent에 전달, 재예약 요청

> ==원칙: 해결안 없는 "충돌 감지만" 카드는 금지. 반드시 실행 가능한 옵션을 포함한다.==

## 법정 기한 D-7/D-1 알림

| 기한 유형 | D-7 액션 | D-1 액션 |
|-----------|---------|---------|
| 사업장현황신고 | 브리핑 카드 + 담당자 알림 | 긴급 Push 알림 |
| 소방 점검 | 체크리스트 초안 생성 (Compliance Agent 요청) | 당일 알림 |
| 어린이통학버스 정기 점검 | 점검 예약 확인 요청 | 긴급 Push 알림 |
| 강사 계약 만료 | Staff Agent에 갱신 검토 요청 | 원장 직접 알림 |
| 현금영수증 발급 기한 | Finance Agent에 처리 요청 | 긴급 Push 알림 |

## 실패/예외 처리

| 상황 | 처리 |
|------|------|
| 대체 강사 후보 없음 | 에스컬레이션 카드 → 원장 직접 판단 |
| 구글 캘린더 동기화 오류 | 감사 로그 기록 + 원장 즉시 알림, 다음 사이클에 재시도 |
| 시험 일정 API 수집 실패 | "수동 확인 필요" 카드 생성, 자동 재시도 3회 |
| 법정 기한 알림 발송 실패 | 대체 채널(SMS)로 재발송, 실패 기록 감사 로그 |
| 브리핑 카드 미수신 | 30분 후 재발송, 미수신 시 원장 전화 알림 |
