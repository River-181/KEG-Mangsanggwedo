# Workflow: 일일 브리핑 (Morning Briefing)

> 매일 07:00 heartbeat — 원장이 아침에 대시보드를 열면 AI 팀이 밤사이 처리한 것을 보고한다

---

## 개요

**트리거**: 매일 07:00 (cron schedule) + 원장 앱 오픈 시 즉시 갱신

**목적**: 원장이 5분 이내에 밤사이 AI 처리 현황을 파악하고 필요한 승인만 완료

**참여 에이전트**: Orchestrator → Complaint + Retention + Scheduler + Analytics (병렬 실행)

---

## 실행 흐름

### 1. Orchestrator 시작 (07:00)
```
Orchestrator.prompt = "어제 밤부터 지금까지 처리한 것을 요약해줘"
Context:
  - 처리 기한: 09:00 (2시간)
  - 예산: 에이전트 병렬 실행만 허용 (순차 실행 금지)
  - 우선순위: Level 1 승인필요 > Level 0 자동처리 확인
```

### 2. 병렬 에이전트 실행 (모두 동시)

| 에이전트 | 수집 데이터 | 산출물 | 시간 제한 |
|---------|----------|--------|----------|
| **Complaint** | 밤새 접수된 민원 | 분류 + 응답 초안 | 90초 |
| **Retention** | 이탈 징후 감지 (로그인/결제) | 위험 학생 리스트 + 개입 안내 | 60초 |
| **Scheduler** | 오늘 일정 (강사/상담/차량) | 예외 사항 플래그 | 45초 |
| **Analytics** | 어제 성과 (수강생 만족도/통과율) | 주요 지표 요약 | 30초 |

## 브리핑 카드 UI 구성

원장이 보게 될 순서:

### Card 1: 미처리 민원 (상단)
```
🔴 승인 필요: 2건
  - [고민 1] 환불 요청 (수강료 50만원) → 정책: 2주 내 환불
    AI 제안: 이번주 목요일 환불 처리
    [승인] [거절] [수정 후 승인]
  
  - [고민 2] 강사 변경 요청
    AI 제안: 보충 강사 김민정, 내일부터 투입
    [승인] [거절] [수정 후 승인]

✅ 자동 처리됨: 5건 (상세 보기)
  - 수강료 연체 알림 → 카카오톡 발송 (2건)
  - 수강료 결제 안내 → SMS 발송 (3건)
```

### Card 2: 이탈 위험 (중단)
```
⚠️ 위험 학생 3명 (이 주에 로그인 0)
  - 이준호 (초4, 영어) → 지난주 문제풀이 저조
    추천 조치: 보충 상담 제안 + 학부모 연락
  
  - 박수진 (중1, 수학)
  - 정민준 (고2, 과학)

[자동 연락 시작하기] [수동으로 관리]
```

### Card 3: 오늘 스케줄 (하단)
```
📅 오늘 일정 (4월 9일)
  ✅ 강사 3명 근무 예정
  ⏰ 13:00 상담반 개강 (예정대로)
  🚗 셔틀 출발 16:30 (날씨: 맑음, 정시 출발 예상)
  
[예외 없음]
```

### Card 4: 법정 기한 (우측 고정)
```
📋 법정 기한 추적
  D-30: 안전교육 예정 (5월 9일)
  D-7: 소방 안전검사 (4월 16일)
```

## Level 0 자동 처리 항목 vs Level 1 승인 필요 항목

### Level 0 (자동 완료, 승인 불필요)
- ✅ 연체 알림 발송 (템플릿 확정 후)
- ✅ 이탈 징후 감지 및 인벤토리 업데이트
- ✅ 오늘 일정 확인 및 예외 플래그
- ✅ 감사 로그 자동 기록

### Level 1 (원장 승인 필요)
- 🔴 환불 실행 (금액 > 100만원 or 정책 외)
- 🔴 강사 변경/해고 (이의 가능)
- 🔴 학부모/학생 연락 (개인정보 포함)
- 🔴 환불 이외 금전 거래
- 🔴 법적 문서 발송 (공지/계약/약관)

## 성공 지표

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| **5분 완료율** | > 80% | 원장이 로그인부터 마지막 승인까지 시간 추적 |
| **미처리 민원 최소화** | Level 1 건수 < 3 | 일일 브리핑 카드 수 |
| **이탈 감지 정확도** | 정밀도 > 85% | 실제 이탈한 학생과 AI 예측 비교 (주 1회) |
| **에러율** | < 1% | AI가 잘못 분류한 민원 수 / 전체 민원 |

---

## 필요 리소스

- **Message Template DB**: 자동 발송 메시지 미리 작성 필요
- **Compliance Rulebook**: 환불 정책, 환전 조건 정의
- **Churn Threshold**: 이탈 판단 기준 (예: 10일 미로그인)
- **Calendar Sync**: Google Calendar 또는 학원 내부 스케줄 시스템 연동
