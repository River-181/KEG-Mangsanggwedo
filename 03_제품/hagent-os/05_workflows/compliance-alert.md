# Workflow: 법정 기한 & 규제 대응

> Compliance Agent가 법정 기한을 추적하고 원장에게 선제적으로 알리는 흐름

---

## 개요

**목적**: 한국 교육 기관이 준수해야 할 법정 기한(소방·세무·노무·안전교육·개인정보)을 놓치지 않도록 자동 추적

**트리거**:
- 매일 09:00 정기 스캔 (모든 기한 재확인)
- 실시간: D-30, D-7, D-1 알림 (각 1회)
- 공문 수신 시 즉시 (확인 및 원장 요약)

**참여 에이전트**: Compliance + Notification + Scheduler

---

## 법정 기한 유형

### 1. 소방 안전
```
- 정기 안전검사: 연 1회 (일정 자유)
- 소방 설비 점검: 분기별 (3/6/9/12월)
- 응급 대응 훈련: 연 2회 이상
- 기록 보관: 3년 유지
```

### 2. 세무 (학원 적용)
```
- 월 부가세 신고: 매월 25일까지
- 원천징수 신고: 매월 10일까지
- 종합소득세: 연 1회 (5월 31일까지)
- 증빙 보관: 5년
```

### 3. 노무 (강사 5명 이상)
```
- 근로계약서: 고용 시 및 매년 갱신
- 4대보험: 고용 후 14일 내 가입
- 급여명세: 월급 지급 시
- 사업보고서: 연 1회 (노동청)
```

### 4. 안전교육 (의무)
```
- 정기 안전교육: 연 2회 (6시간 이상)
- 신입 직원 교육: 고용 후 7일 내
- 기록 보관: 3년
```

### 5. 개인정보 보호
```
- 개인정보 처리 방침 공시: 의무
- 학부모 동의 (마케팅): 연 1회 재확인
- 데이터 삭제 요청 대응: 30일 내
- 개인정보 감시 점검: 분기 1회
```

## D-30 / D-7 / D-1 알림 트리거

```
Compliance.scan()
  ├─ 모든 기한 조회
  └─ 각 기한별로:
      ├─ TODAY가 D-30인가? → 알림: "30일 후 마감"
      ├─ TODAY가 D-7인가?  → 알림: "일주일 남음" (강조)
      ├─ TODAY가 D-1인가?  → 알림: "내일 마감!" (빨강)
      └─ TODAY가 기한일인가? → 알림: "오늘이 마감일입니다" (긴급)
```

### D-30 알림
```
원장 메시지:
"4월 안전교육 일정을 잡아주세요 (5월 9일 마감)"

AI 제안:
- 보험사와 교육 일정 협의 중? (자동 업데이트)
- 지난해 일정 참고: 5월 15일 실행
  → 올해 일정 자동 제안
```

### D-7 알림
```
원장 메시지 (강조):
"⚠️ 안전교육이 1주일 남았습니다. 확인해주세요."

AI 제안:
- 담당자: ?
- 일정 확정: 5월 9일 오전 10시
- 강사/시설 예약 상태: ⏳ 대기중
  → [강사에게 알림] [스케줄 확정]
```

### D-1 알림
```
원장 메시지 (빨강):
"🔴 내일 마감입니다. 즉시 처리가 필요합니다."

AI 제안:
- 소방 검사 예약: 아직 미정
- 권장: 오늘 중으로 전화 예약
  → [자동 전화 발신?] [직접 전화]
```

## 공문 수신 → 해석 → 원장 요약

### 1. 공문 수신 (자동)
```
- 이메일 모니터링 (원장 관리자 이메일)
- 한국법령정보센터 RSS 모니터링
- 지역교육청 공지사항 스크래핑
- 노동청/세무청 알림 (가입 선택)
```

### 2. 공문 해석 (korean-law-mcp 활용)
```
korean-law-mcp.parse({
  document: "새 공문 텍스트",
  domain: "education", // 도메인 지정
  task: "extract_deadline_and_action"
})

결과:
{
  title: "2026년 상반기 안전교육 실시 계획",
  deadline: "2026-05-31",
  actions: [
    "교원 및 직원 안전교육 2시간",
    "학생 생활안전교육 1시간",
    "기록 제출"
  ],
  penalty: "100만원 이하 과태료",
  source_url: "..."
}
```

### 3. 원장 요약 전달
```
Notification.send(academy_head, {
  channel: "app_alert", // 앱 알림 우선
  title: "새로운 규제 공문",
  summary: "상반기 안전교육 의무 (5월 31일까지)",
  action: "요약 보기 & 달력에 추가",
  ai_note: "작년과 동일한 내용. 지난해 일정 참고 가능."
})
```

## 원장 확인 → 조치 기록

### 원장 액션 (대시보드에서)

1. **기한 일정 확정**
   ```
   [안전교육 일정 확정하기]
   - 날짜: [dropdown]
   - 담당자: [select]
   - 대상: 교원(O) / 직원(O) / 학생(O)
   [저장]
   ```

2. **담당자 할당**
   ```
   → 자동으로 담당자(예: 실장)에게 알림
   "4월 9일, 원장이 5월 9일 안전교육 일정을 확정했습니다."
   ```

3. **추적 상태 업데이트**
   ```
   Timeline:
   📌 2026-04-09 원장 승인
   ⏳ 2026-04-15 강사 예약 완료 (담당자 진행)
   ⏳ 2026-05-08 최종 확인
   ✅ 2026-05-09 실행 완료
   ```

## 관련 에이전트

| 에이전트 | 역할 |
|---------|------|
| **Compliance** | 기한 추적, 공문 분석, 정책 해석 |
| **Notification** | 원장·담당자·강사 알림 발송 |
| **Scheduler** | 달력 예약, 일정 동기화 |
| **Admin** | 기록 보관 및 감사 로그 |

| 지표 | 목표 |
