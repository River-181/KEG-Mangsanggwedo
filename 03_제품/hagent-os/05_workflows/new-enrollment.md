# Workflow: 신규 상담 → 등록 전환

> Intake Agent가 주도하는 신규 수강생 유입 → 등록 완료 전 과정 워크플로우.
> 24시간 자동 응답, 상담 예약, 미전환 follow-up, 온보딩까지 커버한다.

---

## 트리거

신규 문의는 다음 세 채널에서 유입된다. 채널과 무관하게 동일한 Intake Agent 파이프라인으로 진입한다.

| 채널 | 트리거 방식 | 자동화 도구 |
|------|------------|-------------|
| **전화** | 수신 콜 → 앱 팝업으로 수동 접수 | 상담사 직접 CRM 입력 |
| **앱/웹폼** | 폼 제출 즉시 on_demand | Intake Agent 자동 응답 |
| **카카오톡** | 카카오채널 메시지 수신 즉시 | kakao-bot-mcp-server |

> 전화 유입은 Intake Agent가 직접 처리할 수 없으므로, 상담사가 CRM에 입력하면 그 시점부터 동일 파이프라인으로 합류한다.

---

## 전체 흐름

```
[문의 유입] (앱/카카오/전화)
    │
    ▼
Intake Agent: 자동 응답 발송 (15분 이내)
  - 인사 + 학원 소개 1~2문장
  - "어떤 과목/학년을 찾고 계신가요?" 정보 수집 시작
    │
    ▼
사전 정보 수집 (챗봇 모드):
  - 학생 학년 / 관심 과목 / 거주 지역
  - 원하는 수업 요일·시간대 (희망 우선순위)
  - 이전 수강 경험 여부
    │
    ▼
Intake Agent: 상담 예약 슬롯 제안
  ← Scheduler Agent에 가용 슬롯 3개 요청
    │
    ▼ [학부모 슬롯 선택]
예약 확정: CRM 등록 + 구글 캘린더 반영
Notification Agent: 예약 확인 문자 발송 + D-1 리마인더 예약
    │
    ▼
[상담 진행] ← 상담사(원장/상담실장) 직접 진행
  - 사전 수집 정보 요약 카드 → 상담사에게 제공
  - 상담 결과 CRM 입력 (전환 / 미전환 / 재연락 필요)
    │
    ├── 전환 결정 → 등록 클로징 단계
    └── 미결정 → follow-up 파이프라인 진입
```

## Intake Agent 역할

### 자동 응답
- 문의 수신 후 **15분 이내** 첫 응답 (목표: 평균 < 15분)
- 응답 내용: 인사 + 간단한 학원 소개 + 첫 질문 1개 (정보 과부하 방지)
- 야간(22시~07시) 응답: "내일 오전 안내드리겠습니다" + 예약 링크 제공

### 정보 사전 수집
- 챗봇 대화로 최대 3~4개 질문으로 마무리 (간결함 유지)
- 수집 데이터: 학년, 과목, 지역, 가용 요일·시간, 목표/관심사
- 수집 완료 시 상담사용 요약 카드 자동 생성

### 강사 추천 (상담 전 참고용)
- 지역 + 과목 기반으로 적합 강사 1~2인 내부 태깅
- `py-mcp-naver` k-skill로 지역 분석 (시/도/구 기반 수강생 풀 파악)
- 최종 배정은 상담 후 원장/실장 확정

### 리마인더 발송
- 예약 확인 즉시: 상담 날짜·시간·장소 안내 문자
- D-1: "내일 상담 일정 확인" 리마인더 (aligo-sms-mcp)

## Scheduler Agent 연계

```
[Intake Agent] → "가용 상담 슬롯 3개 필요 — 학부모 A 요청"
    │
    ▼
Scheduler Agent:
  - 상담실 가용 시간대 조회
  - 배정 상담사 일정 확인 (충돌 제외)
  - 가용 슬롯 3개 추출 → Intake Agent 반환
    │
    ▼
[Intake Agent] → 학부모에게 슬롯 3개 제시
    │
    ▼ [학부모 선택]
Scheduler Agent: 상담 예약 DB 등록 + 캘린더 반영
```

> 상담실이 모두 차 있을 경우: "현재 예약 가능한 슬롯이 없습니다 — 가장 빠른 날짜는 OO일입니다" 안내.

## 상담 후 등록 전환 체크리스트

상담 완료 후 CRM에 결과 입력 시 상담사가 체크하는 항목.

- [ ] 수강 희망 반 확인 (과목 / 요일·시간 / 강사)
- [ ] 반 잔여석 확인 (Scheduler Agent에서 실시간 조회)
- [ ] 수강료 및 납부 방법 안내 완료
- [ ] 교재·준비물 안내 완료
- [ ] 등록 의사 확인 (즉시 등록 / 재고 필요 / 거절)
- [ ] CRM에 상담 결과 기록 (전환 사유 또는 미전환 사유)

> ==승인 지점: 신규 등록 클로징은 원장 또는 상담실장이 직접 확정한다. Intake Agent는 보조만 한다.==

## 미전환 Follow-up (cron: 3일, 7일)

상담 후 등록하지 않은 학부모에게 자동 재연락한다.

```
[상담 완료 — 미등록 상태]
    │
    ├── D+3 (cron): 1차 follow-up
    │   내용: "상담 이후 궁금하신 점이 있으신가요?" + 등록 링크
    │   채널: 카카오 알림톡 (aligo-sms-mcp 백업)
    │
    ├── D+7 (cron): 2차 follow-up
    │   내용: "이번 달 등록 시 OO 혜택 안내" (시즌 맞춤 문구)
    │   채널: 카카오 알림톡
    │
    └── D+7 미응답 시: CRM에 "장기 미전환" 태깅
        Retention Agent에 이관 (필요 시 전화 재연락 제안)
```

**follow-up 발송 제한**
- 동일 학부모에게 최대 3회 (과도한 재촉 방지)
- 거절 의사 명시 시 즉시 중단, CRM에 "거절 처리" 기록
- 야간(22시~07시) 발송 금지

## 등록 완료 → 온보딩

```
[등록 확정 + 수납 완료]
    │
    ▼
Intake Agent: 온보딩 시퀀스 시작

  Step 1. 환영 메시지 + 첫 수업 일정 안내
          채널: 카카오 알림톡
          내용: 강사명 / 강의실 위치 / 준비물 / 주차 안내

  Step 2. 출결 시스템 등록 안내
          앱 설치 링크 + 등하원 알림 설정 방법

  Step 3. 반 배정 확인
          Scheduler Agent: 수강생 반 배정 DB 등록
          해당 강사에게 신규 수강생 알림 카드 발송

  Step 4. 수납 안내
          Billing Agent: 수강료 청구서 발행 (다음 납기일 기준)

  Step 5. D+7 온보딩 체크인
          "첫 수업은 어떠셨나요?" 자동 문자 → 피드백 수집
```

## 실패/이탈 처리

| 상황 | 처리 |
|------|------|
| 응답 15분 초과 | 상담사 알림 카드 발송 ("미응답 문의 있음") |
| 상담 예약 후 노쇼 | D+1 자동 재예약 제안 문자 1회 발송 |
| 등록 후 D+3 내 취소 요청 | 원장 에스컬레이션 카드 (환불 규정 안내 포함) |
| 카카오 채널 메시지 수신 실패 | aligo-sms-mcp SMS로 자동 전환 |
| CRM 입력 누락 | 상담 완료 3시간 후 상담사에게 "결과 입력 필요" 리마인더 |

---

## 성공 지표

| 지표 | 목표 |
|------|------|
| 평균 최초 응답 시간 | < 15분 |
| 문의 → 상담 일정 제안 비율 | ≥ 95% |
| 상담 → 등록 전환율 | ≥ 30% |
| follow-up 후 추가 전환율 | ≥ 10% (미전환 대비) |
| 온보딩 완료율 (앱 설치 기준) | ≥ 80% |

> `lead-conversion-tracker` k-skill이 퍼널 전 단계(문의→상담→등록→재등록)를 자동 추적한다.
