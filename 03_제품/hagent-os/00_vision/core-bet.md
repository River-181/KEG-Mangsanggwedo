# Core Bet

> HagentOS가 거는 하나의 베팅. 모든 후속 문서의 기준점.

---

## Thesis

**"Paperclip의 'company as AI' 개념을 한국 교육에 이식하면, 교육자는 교육 본업에만 집중하고, AI 에이전트 팀이 운영 잡무를 대신 처리하는 제로 휴먼 운영이 가능하다."**

기존 에듀테크가 개별 도구(출결 앱, 수납 앱, 생기부 앱)를 제공한다면, HagentOS는 **교육 기관 전체를 운영하는 AI 조직**을 제공한다.

---

## Problem

한국 교육자는 세계에서 가장 많은 비교육 업무에 시달린다.

### 사교육 운영자 (학원 원장/실장)

- 하루 **3시간** 행정 = **월 300만원 기회비용** (bati.ai)
- 업무의 **70%가 비정형 예외 처리**: 심야 민원, 보강, 환불, 강사 대타, 스케줄 조정
- 출결·수납은 자동화됐지만 **민원·이탈·강사 관리는 100% 수동**
- "결국 카카오톡과 엑셀로 돌아간다"
- 학령인구 **500만 붕괴** (2026) → 학원 생존 경쟁 극심화

### 공교육 교사 (담임)

- 주당 행정 **6~8시간** = **OECD 1위** (TALIS 2024)
- 학부모 민원 스트레스 **56.9%** = 세계 2위 (TALIS 2024)
- 악성 민원 경험 **46.8%** (교사노조연맹, n=4,068)
- 생기부 주당 **5~8시간**, 주말·공휴일 처리 45.6%
- 민원 경로 **84%가 교사 개인 폰** → 일원화 요청 92.1%
- 정서적 소진 **68%** (OECD 최상위)

### 강사/교수자

- 수업 14:00~24:00 + 상담·관리·보고 → 교육 품질 향상 시간 없음
- 강사 역량 = 재등록 결정 **1위 요인(47%)** — 그러나 강사에게는 지원 시스템 없음
- 노란봉투법(2026.03) → 강사 근로자성 분쟁 리스크 증가

## Solution

### 핵심 개념: "교육 기관을 운영하는 AI 팀"

```
기존 도구:     출결 앱 + 수납 앱 + 메신저 + 엑셀 (분절, 수동)
HagentOS:     오케스트레이터 → [민원팀] [이탈방어팀] [신규상담팀] [강사관리팀]
              → 결과물 카드 → 사람이 원클릭 승인
```

- **Not a chatbot**: 대화가 아니라 **직무 지향 자동 처리**
- **Orchestration**: 한 줄 지시 → 여러 에이전트가 병렬 실행
- **Approval-centered**: AI가 초안, 사람이 승인 (Level 0~4)
- **Data asset**: 모든 케이스·승인·후속조치가 누적 자산이 됨

### 제로 휴먼 레벨

| Level | 이름 | 인간 개입 | 예시 |
|-------|------|:---------:|------|
| 0 | 완전 자동 | 0% | 일일 브리핑 생성 |
| 1 | 초안 + 원클릭 승인 | 10% | 민원 응답 초안 → 전송 |
| 2 | 초안 + 편집 후 발송 | 30% | 복잡한 민원 대응 |
| 3 | 분석 + 사람 결정 | 50% | 이탈 위험 학생 상담 판단 |
| 4 | 정보 제공만 | 90% | 법적·감성 판단 |

## Positioning

### 경쟁 구도

```
                    ┌─ 루틴 자동화 ─────────────────── 비정형 예외 처리 ─┐
                    │                                              │
    단일 도구  ─── 클래스업, 온하이                                      │
                    클래스팅                                         │
                                                                   │
    AI 보조    ─── LGU+ 슈퍼스쿨 (생기부)                               │
                    왓퀴즈 (생기부)                                   │
                                                                   │
    AI 팀      ───                                              ★ HagentOS
    (오케스트레이션)                                               (White Space)
```

### 핵심 차별점 3가지

1. **역할 기반 AI 에이전트 오케스트레이션** — 챗봇이 아니라 AI 팀. 민원팀, 이탈방어팀, 강사관리팀이 병렬 실행. 한국형 아카데미에 맞는 **경쟁사 없음.**
2. **데이터 자산화** — 모든 케이스·승인·후속조치가 누적되어 "우리 학원의 운영 지능"이 됨. Palantir 컨셉의 교육 적용.
3. **오픈소스 + 로컬 배포 옵션** — 교육 데이터가 기관 밖으로 나가지 않음. 대형 플랫폼 종속 탈피.

## MVP Bet (이번 대회)

> **"학원 원장이 아침에 대시보드를 열면, AI 팀이 밤사이 민원을 처리하고 이탈 위험 학생을 감지해서 보고한다. 원장은 5분 안에 승인만 하면 된다."**

### 증명할 것

1. 심사위원이 **"단순 챗봇이 아니라 AI 팀이다"**라고 느낀다
2. 2분 데모에서 **에이전트 3개가 병렬 실행**되는 장면이 나온다
3. **확장성**에서 다른 팀과 확연히 차별화된다 (사교육 → 공교육 모드 전환)
4. KEG 주최사(IT 학원)가 **"우리한테 바로 쓸 수 있겠다"**고 반응한다

### 데모 핵심 장면 (2분)

```
[원장] "오늘 민원 처리하고 이번 주 이탈 위험 학생 알려줘"

[에이전트 팀 병렬 실행]
  Complaint Agent: "민원 3건 분류 완료
    → 홍길동 어머니(성적불만): 응답 초안 작성"
  Retention Agent: "이탈 위험: 박민준(이번달 결석 4회)
    → 상담 권고"

[원장 액션]
  민원 초안 확인 → "전송" 클릭 (30초)
  박민준 상담 일정 생성

[임팩트]
  기존: 2시간 / HagentOS: 5분
```

## What We're NOT Building

- 출결·수납 자동화 (클래스업, 온하이 이미 커버 — 레드오션)
- 생기부 단독 (LGU+ 슈퍼스쿨, 왓퀴즈 이미 있음)
- 차량 관리 단독 MVP (IoT 없이 데모 설득력 없음)
- NEIS 직접 연동 (개인정보 규제 — Mock 데이터로 대체)
- 수강생 향 B2C 앱 (교육자/운영자에만 집중)
- 학습 콘텐츠 추천 (다른 팀이 잘 할 것)
