# Core Bet

> HagentOS가 거는 하나의 베팅. 모든 후속 문서의 기준점.

---

## Thesis

**"Paperclip의 'company as AI' 개념을 한국 교육에 이식하면, 교육자는 교육 본업에만 집중하고, AI 에이전트 팀이 운영 잡무를 대신 처리하는 제로 휴먼 운영이 가능하다."**

기존 에듀테크가 개별 도구(출결 앱, 수납 앱, 생기부 앱)를 제공한다면, HagentOS는 **교육 기관 전체를 운영하는 AI 조직**을 제공한다.

---

## Problem

한국 교육자는 세계에서 가장 많은 비교육 업무에 시달린다.

### 사교육 운영자 (학원 원장/실장)

- 하루 **3시간** 행정 = **월 300만원 기회비용** (bati.ai)
- 업무의 **70%가 비정형 예외 처리**: 심야 민원, 보강, 환불, 강사 대타, 스케줄 조정
- 출결·수납은 자동화됐지만 **민원·이탈·강사 관리는 100% 수동**
- "결국 카카오톡과 엑셀로 돌아간다"
- 학령인구 **500만 붕괴** (2026) → 학원 생존 경쟁 극심화

### 공교육 교사 (담임)

- 주당 행정 **6~8시간** = **OECD 1위** (TALIS 2024)
- 학부모 민원 스트레스 **56.9%** (TALIS 2024 반올림 57%) = 세계 2위
- 악성 민원 경험 **46.8%** (교사노조연맹, n=4,068)
- 생기부 주당 **5~8시간**, 주말·공휴일 처리 45.6%
- 민원 경로 **84%가 교사 개인 폰** → 일원화 요청 92.1%
- 정서적 소진 **68%** (OECD 최상위)

### 강사/교수자

- 수업 14:00~24:00 + 상담·관리·보고 → 교육 품질 향상 시간 없음
- 강사 역량 = 재등록 결정 **1위 요인(47%)** — 그러나 강사에게는 지원 시스템 없음
- 노란봉투법(2026.03) → 강사 근로자성 분쟁 리스크 증가

## Solution

### 핵심 개념: "교육 기관을 운영하는 AI 팀"

```
기존 도구:     출결 앱 + 수납 앱 + 메신저 + 엑셀 + 구글 캘린더 (분절, 수동)
HagentOS:     오케스트레이터 → [민원팀] [이탈방어팀] [스케줄러] [강사관리팀] [법무팀]
              → 통합 캘린더 + 결과물 카드 → 사람이 원클릭 승인
              → 모바일에서도 알림 수신 + 승인
```

- **Not a chatbot**: 대화가 아니라 **직무 지향 자동 처리**
- **Orchestration**: 한 줄 지시 → 여러 에이전트가 병렬 실행
- **Schedule-aware**: 에이전트가 강사·차량·상담·법정기한 **스케줄 맥락을 알고** 판단 (구글 캘린더 연동)
- **k-skill powered**: 에이전트가 한국형 스킬(법규 조회, 환불 계산, 최저임금 API, 교육 통계 등)을 장착하여 실행
- **Approval-centered**: AI가 초안, 사람이 승인 (Level 0~4). 모바일 반응형 — 원장은 휴대폰으로 승인
- **Data asset**: 모든 케이스·승인·후속조치가 누적 자산이 됨. 엑셀 import/export로 기존 데이터 즉시 활용

### 제로 휴먼 레벨

| Level | 이름 | 인간 개입 | 예시 |
|-------|------|:---------:|------|
| 0 | 완전 자동 | 0% | 일일 브리핑 생성 |
| 1 | 초안 + 원클릭 승인 | 10% | 민원 응답 초안 → 전송 |
| 2 | 초안 + 편집 후 발송 | 30% | 복잡한 민원 대응 |
| 3 | 분석 + 사람 결정 | 50% | 이탈 위험 학생 상담 판단 |
| 4 | 정보 제공만 | 90% | 법적·감성 판단 |

## Positioning

### 경쟁 구도

```
                    ┌─ 루틴 자동화 ─────────────────── 비정형 예외 처리 ─┐
                    │                                              │
    단일 도구  ─── 클래스업, 온하이                                      │
                    클래스팅                                         │
                                                                   │
    AI 보조    ─── LGU+ 슈퍼스쿨 (생기부)                               │
                    왓퀴즈 (생기부)                                   │
                                                                   │
    AI 팀      ───                                              ★ HagentOS
    (오케스트레이션)                                               (White Space)
```

### 핵심 차별점 4가지

1. **역할 기반 AI 에이전트 오케스트레이션** — 챗봇이 아니라 AI 팀. 민원팀, 이탈방어팀, 강사관리팀, 법무팀, 재무팀이 병렬 실행. 한국형 아카데미에 맞는 **직접 경쟁 제품이 아직 없는 영역.**
2. **k-skill 생태계** — 한국 교육 도메인에 특화된 스킬 패키지를 GitHub 기반으로 설치·공유. 최저임금 조회, 환불 계산, 법규 검색([korean-law-mcp](https://github.com/pyhub-kr/korean-law-mcp)), 교육 공공데이터 연동, HWP 처리 등을 커뮤니티가 계속 추가. **에이전트는 범용이지만 스킬은 한국형** — 이것이 진짜 해자(moat).
3. **데이터 자산화** — 모든 케이스·승인·후속조치가 누적되어 "우리 학원의 운영 지능"이 됨. 엑셀 import/export로 기존 데이터 즉시 활용 + 운영 리포트 자동 생성. Palantir 컨셉의 교육 적용.
4. **오픈소스 + 로컬 배포 옵션** — 교육 데이터가 기관 밖으로 나가지 않음. 대형 플랫폼 종속 탈피.

## MVP Bet (이번 대회)

> **"학원 원장이 아침에 대시보드를 열면, AI 팀이 밤사이 민원을 처리하고, 이탈 위험 학생을 감지하고, 내일 강사 공결에 대한 대체를 준비해서 보고한다. 원장은 휴대폰으로 5분 안에 승인만 하면 된다."**

### 증명할 것

1. 심사위원이 **"단순 챗봇이 아니라 AI 팀이다"**라고 느낀다
2. 2분 데모에서 **에이전트 3개가 병렬 실행**되는 장면이 나온다
3. **통합 스케줄러**로 "이건 진짜 운영 OS다"라고 느낀다 (강사·상담·차량·법정기한)
4. **k-skill 레지스트리**로 "이건 앱이 아니라 플랫폼이다"라고 인식한다
5. **확장성**에서 다른 팀과 확연히 차별화된다 (사교육 → 공교육 모드 전환)
6. KEG 주최사(IT 학원)가 **"우리한테 바로 쓸 수 있겠다"**고 반응한다

### 데모 핵심 장면 (2분)

```
[원장] "오늘 민원 처리하고 이번 주 이탈 위험 학생 알려줘"

[에이전트 팀 병렬 실행]
  Complaint Agent: "민원 3건 분류 완료
    → 홍길동 어머니(성적불만): 응답 초안 작성"
  Retention Agent: "이탈 위험: 박민준(이번달 결석 4회)
    → 상담 권고"

[원장 액션]
  민원 초안 확인 → "전송" 클릭 (30초)
  박민준 상담 일정 생성

[임팩트]
  기존: 2시간 / HagentOS: 5분
```

### 왜 k-skill이 핵심인가

1. **확장성**: 에이전트 2개(민원+이탈)만으로는 "앱"이지 "플랫폼"이 아님. k-skill이 있어야 플랫폼
2. **해자(moat)**: 범용 AI는 한국 학원법, 교습비 환불 기준, 소방시설법 개정을 모름. k-skill은 이걸 패키징
3. **생태계가 이미 존재**: korean-law-mcp, aligo-sms, portone, py-mcp-naver 등 한국 서비스용 MCP가 이미 활발. HagentOS는 이걸 **교육 맥락으로 엮는 오케스트레이터** 역할
4. **커뮤니티**: 원장·개발자가 자기 학원에 맞는 스킬을 만들어 공유 → 네트워크 효과
5. **도메인 팩**: 보습→태권도→운전면허→공교육 확장이 "스킬 추가"로 가능

## What We're NOT Building

- 출결·수납 자동화 (클래스업, 온하이 이미 커버 — 레드오션)
