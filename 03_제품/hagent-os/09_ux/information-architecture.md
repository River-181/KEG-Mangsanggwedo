# Information Architecture

> HagentOS 화면 구성 전체 지도. 개발 시 라우트와 레이아웃 기준.

---

## 레이아웃 구조 (4-Zone)

### 데스크톱

| Zone | 너비 | 역할 |
|------|------|------|
| Zone 0: 기관 레일 | 72px | 기관 전환, 알림 인디케이터 |
| Zone 1: 사이드바 | 240px (접힘 가능) | 전체 네비게이션 |
| Zone 2: 메인 콘텐츠 | flex-1 | 페이지 본문 |
| Zone 3: 속성 패널 | 320px (태블릿 이하 숨김) | 컨텍스트별 상세 속성 |

### 모바일

- 하단 탭바 5개 (고정)
- 사이드바: 스와이프로 열리는 오버레이

**CommandPalette**: Cmd+K 글로벌 검색/네비게이션 오버레이 (모든 화면에서 접근 가능)
- 케이스 검색, 에이전트 이동, 페이지 이동
- 키보드: ↑↓ 탐색, Enter 이동, Esc 닫기

---

## 사이드바 구조

사이드바 항목 순서 (위→아래):

1. **기관명 + 검색 버튼**
2. **케이스 등록 버튼** — 새 케이스 인라인 생성
3. **대시보드** — live count 배지
4. **알림함** — unread 배지, 실패 알림 시 danger 표시
5. **Work 섹션**
   - 케이스 목록
   - 자동 실행
   - 운영 목표
6. **운영 영역 섹션** (접힘, 영역 그룹 목록)
7. **에이전트 팀 섹션** (접힘, 각 에이전트: 이름 + live 인디케이터)
8. **기관 관리**
   - 에이전트 조직도
   - k-skill 레지스트리
   - API 예산
   - 처리 이력
   - 설정
9. **푸터** — 버전, 다크·라이트 토글

## 지시/프롬프트 인터페이스 (Paperclip 핵심 패턴)

==Paperclip의 가장 중요한 UX — 원장이 자연어로 에이전트에게 일을 시키는 창==

**진입점 (3곳)**:
1. **대시보드 커맨드 바** — 대시보드 상단에 "오케스트레이터에게 지시하기" 입력 창
2. **케이스 생성 시 지시** — 새 케이스 등록 + 에이전트 배정 + 자연어 지시 통합
3. **에이전트 상세 → 태스크 지시** — 특정 에이전트에 직접 작업 지시

**흐름**:
```
원장 입력: "오늘 들어온 민원 정리하고 이번 주 이탈 위험 학생 보여줘"
    ↓
Orchestrator 수신 → 계획 수립
    ↓
Complaint Agent + Retention Agent 병렬 디스패치
    ↓
각 에이전트 결과 → 케이스/승인 카드 생성
    ↓
원장에게 알림 (승인 큐 or 대시보드)
```

**UI 상세**:
- 입력 창: textarea + "실행" 버튼 + 에이전트 멘션 (`@민원담당`)
- 실행 중: 프로그레스 인디케이터 + "분석 중..." 텍스트
- 결과: 생성된 케이스/승인 카드 링크

---

## 케이스 상세 화면 (3존)

==가장 복잡한 핵심 화면==

**Zone 2 — 메인**
- 케이스 제목, 본문, 첨부파일
- **라이브 에이전트 실행 결과** (Paperclip 핵심):
  - `Live Case runs (N)` 섹션 — 실행 중인 AgentRun 표시
  - "Waiting for run output..." + 실시간 트랜스크립트 피드
  - `[Stop]` `[Run 열기]` 버튼
  - 완료 시: 출력 텍스트 (초안, 분석 결과 등) + 토큰 사용량
- 탭: 댓글 / 하위 케이스 / 처리 이력

**Zone 3 — 속성 패널**
- 상태, 우선순위, 유형
- 담당 에이전트
- 운영 영역 (프로젝트)
- 라벨
- 생성일, 갱신일

---

## 대시보드 화면

1. **지시 입력 바** — "오케스트레이터에게 지시하기" 텍스트 입력 (핵심 CTA)
2. **라이브 에이전트 패널** — 현재 실행 중인 에이전트 카드 그리드
   - 각 카드: 펄스 점 + 에이전트명 + "Live now" / "N분 전 완료"
   - 연결된 케이스 제목 표시
   - 라이브 트랜스크립트 미리보기
3. **4개 지표 카드** — 활성 에이전트, 진행 중 케이스, 승인 대기, 월 토큰 비용
4. **최근 활동 + 최근 케이스** 2열 그리드
5. **이탈 위험 학생** 요약 (Retention Agent 결과)
6. **오늘 스케줄 요약**

## 처리 이력 (Activity) 화면

> Paperclip Activity 패턴: 모든 에이전트 행동의 감사 로그 타임라인

**라우트**: `/[org]/activity`

**UI**:
- 타입 필터 드롭다운: All types / case / agent / approval / run / comment
- 타임라인 형식 (최신 순):
  - `[에이전트 아이콘] [에이전트명] [행동] on [케이스 코드] — [케이스 제목] [시간]`
- 행동 유형:
  - `checked out` — 에이전트가 케이스를 체크아웃해 작업 시작
  - `commented on` — 에이전트가 케이스에 코멘트 추가
  - `changed status` — 상태 변경 (예: backlog → todo, in_progress → done)
  - `updated document` — 문서 업데이트
  - `issue read marked` — Board(원장)가 이슈를 읽음 표시
- 각 항목 클릭 → 해당 케이스 상세 페이지로 이동

**Paperclip 예시** (이미지 참고):
```
Scriptwriter checked out COC-4 — 유튜브 스크립트 작성    2m ago
Content Manager commented on COC-2 — 콘텐츠 파이프라인    2m ago
Content Manager changed status to in progress on COC-2    2m ago
Board updated document on COC-3 — AI/IT 뉴스 리서치       4m ago
```

## 화면 간 주요 이동 흐름

### 0. 지시 → 실행 → 승인 (핵심 패턴)

```
대시보드 지시 바 → "오늘 민원 정리해줘" 입력
    → Orchestrator 실행 (라이브 인디케이터)
    → 케이스 자동 생성 + 에이전트 배정
    → 승인 큐에 카드 도착
    → 원장 원클릭 승인
```

### 1. 아침 브리핑

```
대시보드 → 승인 큐 → 케이스 상세 → 승인
```

### 2. 민원 처리

```
알림함 → 케이스 상세 → (라이브 실행 결과 확인) → 승인 / 편집 / 반려
```

### 3. 이탈 감지

```
대시보드 이탈 경고 → 케이스 상세 → 상담 일정 생성 → 스케줄 캘린더
```

### 4. 스킬 설치

```
k-skill 레지스트리 → 스킬 상세 → 에이전트에 장착
```

## 실시간 이벤트 (WebSocket)

HagentOS는 WebSocket으로 실시간 상태를 스트리밍한다 (Paperclip `LiveEvent` 패턴).

| 이벤트 | 트리거 | UI 반응 |
|--------|--------|---------|
| `agent.run.started` | 에이전트 실행 시작 | 사이드바 live dot 표시 |
| `agent.run.completed` | 실행 완료 | 대시보드 갱신, 토스트 알림 |
| `case.created` | 케이스 생성 | 목록 즉시 업데이트 |
| `approval.created` | 승인 요청 생성 | 알림함 배지 증가 |
| `budget.incident` | 예산 초과 | 빨간 배너 표시 |

연결: `ws(s)://{host}/api/organizations/{orgId}/events/ws`
재연결: 지수 백오프 (max 15s)
