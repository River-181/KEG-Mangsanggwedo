# Information Architecture (정보 구조)

## 전체 네비게이션 구조

```
HagentOS
├── 대시보드 (/)                   ← 진입점, 오전 브리핑 + 승인 큐
│   ├── 승인 카드 큐 (/approvals)
│   │   └── 케이스 상세 (/approvals/:id)
│   ├── 스케줄 캘린더 (/schedule)
│   └── 이탈 위험 알림 (인라인)
├── k-skill 레지스트리 (/skills)   ← 설치·장착·관리
│   ├── 스킬 상세 (/skills/:slug)
│   └── 에이전트에 장착 (모달)
├── 에이전트 관리 (/agents)
│   ├── 에이전트 상세 (/agents/:id)
│   └── 실행 히스토리 (/agents/:id/runs)
├── 온보딩 (/onboarding)           ← 최초 1회, 3분 이내
└── 설정 (/settings)
    ├── 학원 프로필
    ├── 알림 설정
    └── 연동 관리
```

## 핵심 화면 목록

| 화면 | URL 패턴 | 역할 | MVP |
|------|----------|------|-----|
| 대시보드 | `/` | 오전 브리핑, 승인 큐, 오늘 스케줄 요약 | ✅ |

| 스케줄 캘린더 | `/schedule` | 학원 수업·상담·행사 일정, 에이전트 예약 작업 | ✅ |
| k-skill 레지스트리 | `/skills` | 스킬 검색, 설치, 에이전트 장착 | 🔲 |
| 에이전트 관리 | `/agents` | 에이전트 목록, 상태, ON/OFF | 🔲 |

| 실행 히스토리 | `/agents/:id/runs` | 에이전트 과거 실행 기록, 성과 지표 | 🔲 |

## 화면 간 이동 흐름

### 1. 아침 루틴 (핵심 여정)
```
푸시 알림 수신
  → 대시보드 (승인 큐 배지 확인)
    → 승인 카드 큐 (카드 목록)
      → 케이스 상세 (내용 확인)
        → 승인 / 수정 후 승인 / 반려
          → 대시보드 복귀
```

### 2. 스킬 설치 여정
```
대시보드 또는 에이전트 상세
  → k-skill 레지스트리 (스킬 검색)
    → 스킬 상세 (설명·평점 확인)
      → 설치 완료 → 에이전트 장착 (모달)
        → 에이전트 관리 복귀
```

### 3. 일정 확인 여정
```
대시보드 (오늘 스케줄 섹션)
  → 스케줄 캘린더 (전체 뷰)
    → 일정 상세 (탭)
      → 에이전트 예약 작업 확인
```

### 4. 온보딩 → 첫 승인
```
첫 접속
  → 온보딩 (학원 유형 선택)
    → 추천 에이전트 팀 확인·수락
      → 에이전트 첫 실행 대기
        → 대시보드 (첫 승인 카드 등장)
```

## 모바일 vs 데스크톱 차이

### 모바일 (원장 기본 디바이스)
- 하단 탭바 4개: 대시보드 / 승인 / 스케줄 / 더보기
- 승인 카드: 풀스크린 카드 스택, 스와이프 제스처 지원
- 에이전트·스킬 관리: "더보기" 탭에서 접근 (자주 안 씀)
- 텍스트 수정: 인라인 편집 (모달 최소화)
- 알림 → 탭 → 승인 3-tap 플로우 최우선

### 데스크톱 (선택적 사용)
- 사이드바 전체 메뉴 상시 노출
- 에이전트·스킬·히스토리 상세 관리 용이
- 승인 카드 + 케이스 상세 2-column 동시 뷰
- 대량 작업 (일괄 승인, 스킬 일괄 장착) 지원

## 데이터 계층

```
Organization (학원)
├── 메타: 이름, 유형, 연동 채널 (카카오톡, 문자 등)
├── Agents (에이전트)
│   ├── 메타: 이름, 역할, 상태(active/paused/error)
│   ├── Skills[] (장착된 k-skill 목록)
│   ├── Cases (승인 대기 케이스)
│   │   ├── 상태: pending / approved / edited / rejected
│   │   ├── 내용: 제안 텍스트, 증빙 데이터, 긴급도
│   │   └── 액션 로그: 승인자, 타임스탬프, 수정 내역
│   └── Runs (실행 기록)
│       ├── 트리거: 스케줄 / 이벤트 / 수동
│       ├── 결과: 성공 / 실패 / 대기
│       └── 토큰·비용 소모량
└── Schedules (학원 일정)
    ├── 수업 일정
    ├── 상담 예약
    └── 에이전트 예약 실행
```
