# UX Concepts (UX 컨셉)

## 핵심 UX 원칙

| 원칙 | 설명 |
|------|------|
| **원장 시간 최소화** | 하루 승인 업무 5분 이내 처리 가능하도록 설계 |
| **승인 중심 OS** | 채팅이 아닌 컨트롤 패널 — AI가 제안하고 사람이 결정 |
| **AI가 일하고 사람이 결정** | 에이전트는 배경에서 실행, 원장은 결과만 검토·승인 |
| **모바일 우선** | 핵심 액션 (승인·반려·수정)은 폰 한 손으로 완결 |
| **맥락 있는 알림** | "3명 상담 요청 대기" — 숫자가 아닌 의미 전달 |

## 승인 카드 컴포넌트

### 카드 해부도
```
┌─────────────────────────────────────┐
│ [에이전트 아이콘]  상담 자동 응답 초안  [긴급] │  ← 헤더
│─────────────────────────────────────│
│ 김민준 학생 학부모가 수학 레벨 테스트   │  ← 요약 (2줄)
│ 일정을 문의했습니다. 응답 초안 확인 요망 │
│─────────────────────────────────────│
│ 상담봇 에이전트  ·  2분 전            │  ← 메타
│─────────────────────────────────────│
│  [✕ 반려]   [✎ 수정 후 승인]  [✓ 승인] │  ← 액션 버튼
└─────────────────────────────────────┘
```

### 카드 상태
| 상태 | 색상 신호 | 설명 |
|------|-----------|------|
| `pending` | 흰 배경, 파란 테두리 | 원장 검토 대기 중 |
| `approved` | 연초록 배경 | 원장이 승인 완료, 에이전트 실행 |
| `edited` | 연노랑 배경 | 원장이 내용 수정 후 승인 |
| `rejected` | 연빨강 배경 | 원장이 반려, 에이전트에 피드백 전달 |

### 긴급도 배지
- `긴급` (빨강) — 당일 마감 또는 학부모 대기 중
- `오늘` (주황) — 오늘 처리 권장
- `일반` (회색) — 여유 있음, 배지 없음

## 대시보드 레이아웃

```
┌─────────────────────────────┐
│  오전 브리핑                   │  ← 날짜, 한 줄 요약 ("오늘 AI팀이 12건 처리했어요")
│─────────────────────────────│
│  승인 대기  [3]               │  ← 섹션 헤더 + 배지
│  [카드 1 미리보기]             │
│  [카드 2 미리보기]             │
│  전체 보기 →                  │
│─────────────────────────────│
│  오늘 스케줄                   │  ← 오늘 수업·상담 목록 (3건 이내)
│  14:00 김민준 수학 상담         │
│  16:00 레벨 테스트 3명          │
│─────────────────────────────│
│  ⚠ 이탈 위험  [2명]            │  ← 빨간 강조, 탭하면 학생 목록
└─────────────────────────────┘
```

- 섹션 순서: 긴급도 순 (이탈 위험 → 승인 대기 → 스케줄)
- 데스크톱: 3-column 그리드로 동시 노출

## 에이전트 실행 상태 시각화

### "AI 팀이 일하고 있습니다" 위젯
```
┌─────────────────────────────┐
│  ● 상담봇    응답 초안 작성 중    │
│  ● 출결봇    오늘 결석 집계 중    │
│  ◌ 레포트봇  대기 중             │
└─────────────────────────────┘
```
- `●` 펄싱 애니메이션 (실행 중)
- `◌` 회색 (대기 또는 일시정지)
- `✓` 초록 체크 (완료, 3초 후 사라짐)
- 에이전트 탭 시 실행 상세 드로어 열림

## k-skill 레지스트리 UI

### 설치 → 장착 플로우
```
레지스트리 목록 (카드 그리드)
  → 스킬 상세 (설명, 사용 학원 수, 별점, 가격)
    → [설치] 버튼 탭
      → 설치 완료 토스트 + "에이전트에 장착하기" CTA
        → 에이전트 선택 드로어
          → 장착 완료 → 에이전트 상세 복귀
```

### 활성 스킬 표시
- 에이전트 카드에 스킬 아이콘 최대 3개 노출, 초과 시 "+N"
- 스킬 비활성화: 에이전트 상세 → 스킬 목록 → 토글 OFF

## 모바일 핵심 패턴

### 3-tap 승인 플로우
```
1. 푸시 알림 수신 → 탭
2. 카드 큐 또는 케이스 상세 자동 열림
3. [승인] 버튼 탭 → 완료
```

### 스와이프 제스처
- 오른쪽 스와이프 → 승인 (초록 확인 아이콘)
- 왼쪽 스와이프 → 반려 (빨강 X 아이콘)
- 위로 스와이프 → 케이스 상세 열기
- 스와이프 도중 취소 가능 (50% 미만 시 복귀)

**적용 범위**: Level 0~1 케이스만 (일반 민원, 출결 알림). Level 2+ (환불·법무·강사 해고 등) 은 swipe 비활성화 — 반드시 내용 확인 후 버튼 탭.

### 하단 탭바
```
[ 🏠 홈 ] [ ✓ 승인 ] [ 📅 스케줄 ] [ ⋯ 더보기 ]
```
- 승인 탭에 미처리 건수 배지 상시 표시

## 온보딩 UX

### 목표: 3분 이내 첫 에이전트 실행

```
Step 1/3  학원 유형 선택
  [입시 학원] [영어 학원] [예체능] [보습 학원] [기타]

Step 2/3  추천 에이전트 팀 확인
  ┌─────────────────────┐
  │ 입시 학원 스타터 팀      │
  │ · 상담봇              │
  │ · 출결봇              │
  │ · 성적 분석봇          │
  │ [이 팀으로 시작하기]     │
  └─────────────────────┘

Step 3/3  카카오톡 채널 연동 (선택)
  → 완료 → 대시보드 (첫 에이전트 실행 시작)
```

- 연동은 스킵 가능, 나중에 설정에서 추가
- 온보딩 완료 후 "첫 승인 카드가 곧 도착해요" 안내 토스트

## 빈 상태 (Empty State)

### 첫 접속 (에이전트 미설정)
```
[AI 팀 일러스트]
아직 에이전트가 없어요.
추천 팀을 골라서 지금 바로 시작하세요.
[에이전트 팀 고르기 →]
```

### 에이전트 있지만 처리 건 없음
```
[체크 아이콘]
오늘은 AI 팀이 처리할 일이 없어요.
승인 대기 항목이 생기면 알려드릴게요.
```

- 빈 상태에도 항상 다음 액션 CTA 포함
- 일러스트는 가볍고 친근한 톤 (학원 원장 대상)

## 디자인 방향: 운영 제어판 UX 패턴 + 한국형 재설계

### 운영 제어판 중심 구조

HagentOS는 채팅 앱이 아닌 **운영 제어판(control plane)** 중심으로 설계되었다. 이를 통해 원장은 효율적으로 에이전트의 제안을 검토하고 승인 결정을 내릴 수 있다.

| UX 패턴                           | HagentOS 구현                             |
| --------------------------------- | ------------------------------------- |
| **대시보드 진입점**                | 대시보드가 기본 화면, 채팅 인터페이스 없음                |
| **카드 기반 케이스 큐**              | 승인 대기 카드 큐 (pending → approved/rejected 상태 전환) |
| **에이전트 팀 시각화**               | 사이드바 에이전트 패널 (역할별 상담봇·출결봇·레포트봇 표시)     |
| **승인 게이트 플로우**               | 원장 3-버튼 승인 (승인·수정·반려)                 |
| **활동 로그 & 감시 추적**           | 에이전트 활동 타임라인 (누가·언제·무엇을 처리했는지 기록)    |
| **계층적 의사결정**                | 학원 운영 목표 → 케이스 자동 생성 → 원장 승인 흐름        |

### 한국 원장 친화 스킨 (차별화)

**색상 체계**
- 메인: 청록/민트 계열 (`#0D9488` ~ `#14B8A6`) — 신뢰·전문성 전달
- 배경: 따뜻한 흰색 (`#FAFAF9`) — 차가운 기업용 회색 지양
- 강조: 카카오톡 스타일 빨간 뱃지 (`#FF3B30`) — 한국인에게 익숙한 알림 신호

**타이포그래피**
- 본문: Noto Sans KR 400/500, 16px 이상 — 50대 원장 가독성 고려
- 라벨: 크고 명확, 약어·영어 전문용어 최소화
- 언어: 100% 한국어 UI (설정 메뉴 포함)

**터치 최적화**
- 모든 터치 대상: 최소 48px (WCAG AA 기준)
- 카드 크기: 모바일에서 엄지 한 손 조작 가능
- 대시보드: 카드 그리드 (데스크톱 3열 / 모바일 1열)

**아이콘 & 시각 언어**
- 학원 맥락 이모지 병용: 📋 민원, ⚠️ 이탈 위험, 📅 스케줄, 💬 상담, 📊 성적
- 추상적 아이콘보다 즉시 의미 파악 가능한 표현 우선
- 알림 배지: 카카오톡 빨간 원형 뱃지 스타일

**승인 버튼 색상 코딩**
- ✅ 초록 `승인` — 즉시 실행 허가
- ✏️ 회색 `수정 후 승인` — 내용 편집 후 실행
- ❌ 빨강 `반려` — 거부, 에이전트에 피드백

### HagentOS 고유 설계 결정

| 설계 영역 | HagentOS의 선택 |
|------|----------|
| **태스크 모델** | 카드 기반 승인 큐 — 코드/기술 개념 없이 학원 운영만 중심 |
| **대상 사용자** | 비개발자 원장 → UI/UX 전 영역에서 기술 용어 완전 제거, 한국어만 사용 |
| **주요 디바이스** | 모바일 우선 — 원장은 폰으로 현장(교실/상담실)에서 운영, 데스크톱은 보조 |
| **온보딩 속도** | 학원 유형 3-클릭 → 추천 팀 자동 구성 → 3분 내 첫 에이전트 실행 |
| **알림 및 아이콘** | 카카오톡 스타일 빨간 뱃지 + 이모지 (📋 민원, ⚠️ 이탈 위험 등) — 한국인에게 즉시 의미 파악 |
| **비용 투명성** | 토큰 사용량을 한국어 요금으로 요약 ("이번 달 AI 비용 3만원 절약") |

## 에러 / 실패 UX

### 에이전트 실행 실패
```
[에이전트 카드 — 주황 테두리]
⚠ 상담봇이 응답을 보내지 못했어요.
원인: 카카오 API 일시 오류 (503)
[재시도] [자세히 보기]
```
- 원장에게 기술 에러코드 직접 노출 금지
- 원인을 한국어 한 줄로 요약
- 재시도 버튼 항상 제공

### API 오류 (외부 서비스)
- 토스트 메시지: "카카오 연동에 잠깐 문제가 생겼어요. 자동으로 재시도할게요."
- 3회 재시도 후에도 실패 시 원장에게 알림 + 수동 처리 안내

### 예산 초과
```
[배너 — 빨간 배경]
이번 달 AI 사용량이 한도에 도달했어요.
에이전트가 일시 중지되었습니다.
[한도 늘리기] [이번 달 사용 내역]
```
- 한도 80% 도달 시 사전 경고 알림 발송
- 에이전트 중지 전 원장 승인 요청
