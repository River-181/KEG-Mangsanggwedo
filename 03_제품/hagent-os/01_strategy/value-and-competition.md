## Value Proposition

### For 학원 운영자 (MVP)

```
현재:  카카오톡 + 클래스업(출결) + 엑셀 + 머릿속
       → 민원 3건 직접 답장 (30분)
       → 출결 이상 파악 (15분)
       → 강사 스케줄 조정 (20분)
       → 교육청 공문 해석 (모름 → 무시)
       → 합계: 하루 3시간 / 월 300만원 기회비용

HagentOS:  아침 대시보드 열기
           → "민원 3건 처리 완료(승인 대기), 이탈 위험 학생 1명, 강사 A 성과 하락"
           → 승인 3번 클릭 → 5분 완료
           → 특이사항 없는 날: 완전 자동 (Level 0)
```

### 가치 제안 한 문장

> **"학원의 비정형 운영 업무를 AI 에이전트 팀이 처리하고, 원장은 승인만 합니다."**

### Value Canvas

| 항목          | 기존 (출결/수납 도구)       | HagentOS                |
| ----------- | ------------------- | ----------------------- |
| **해결하는 업무** | 출결, 수납, 알림 (루틴 30%) | 민원, 이탈, 강사, 스케줄, 환불, 법무 (예외 70%) |
| **스케줄 관리** | 없음 또는 단순 달력         | 통합 스케줄러 (강사·상담·차량·법정기한) + 구글 캘린더 연동 |
| **AI 역할**   | 없음 또는 단일 기능         | 역할 기반 에이전트 팀 (스케줄 맥락 인지) |
| **데이터 활용**  | 개별 기록               | 누적 맥락 → 자산화 + 엑셀 import/export |
| **스킬 생태계** | 없음                   | k-skill 레지스트리 (법규·세무·노무·환불·공공데이터) |
| **확장성**     | 도메인 고정              | k-skill 도메인 팩 + 모드 전환    |
| **의사결정**    | 사람이 전부              | AI 초안 → 사람 승인           |
| **배포**      | SaaS only           | 오픈소스 + 로컬 옵션            |

## Competitive Landscape

### 경쟁사 매트릭스

| 솔루션 | 주요 기능 | 강점 | 한계 | AI 수준 |
|--------|-----------|------|------|---------|
| **클래스업** | 출결, 수납, 키오스크 | 무료, 3,000+ 학원 도입 | 민원·이탈·강사 관리 없음 | 없음 |
| **온하이** | 출결, 수납, 청구서 | 예체능 학원 특화 | 비정형 예외 처리 없음 | 없음 |
| **클래스팅** | 학부모 소통, 공지, 과제 | 커뮤니케이션 UX | 운영 관리(수납/환불) 약함 | 기본 |
| **아카데밍** | 통합 학원 운영 | 올인원 | 메뉴 복잡, 예외 처리 느림 | 없음 |
| **학원조아** | 학원 관리 | 중소 학원 대상 | 제한적 기능 | 없음 |
| **LGU+ 슈퍼스쿨** | 생기부 AI, 출결 | B2G 계약, 생기부 AI | 단일 기능, 학원 미지원 | 생기부 특화 |
| **왓퀴즈** | 생기부 AI 초안 | 생기부 특화 | 단일 기능 | 생기부 특화 |
| **bati.ai** | 학원 운영 분석 | 데이터 인사이트 | 실행(처리)은 없음 | 분석 |

### 범용 도구 (간접 경쟁)

| 솔루션 | 교육 운영에 사용하는 방식 | 강점 | 한계 |
|--------|------------------------|------|------|
| **Notion** | 학원 DB, 학생 관리, 업무 보드 구축 | AI 기능 내장, 유연한 구조화 | 모든 세팅을 **수동으로** 해야 함. 도메인 지식 없음. 자동 실행/승인 없음 |
| **Google Sheets + Apps Script** | 출결/수납 자동화, 보고서 생성 | 무료, 익숙함 | 파편화, 유지보수 어려움, AI 없음 |
| **카카오톡** | 학부모 소통, 민원 접수 | 전국민 사용 | 케이스 추적 불가, 맥락 누적 안 됨, 원장 번아웃 원인 |
| **ChatGPT / Claude (단독)** | 민원 답변 초안, 공문 요약 | 범용 AI 능력 | 학원 맥락 없음, 워크플로우 없음, 승인 체계 없음, 데이터 축적 안 됨 |

> **핵심 차이**: 범용 도구는 "도구"를 주고 사람이 조립해야 한다. HagentOS는 "이미 조립된 AI 팀"을 제공한다.

| 영역 | 기존 커버 | HagentOS |
|------|-----------|----------|
| 역할 기반 AI 에이전트 오케스트레이션 | **없음** | 핵심 기능 |

| 재등록/이탈 통합 예측 | **없음** | Retention Agent |
| 신규 상담 → 등록 파이프라인 | **없음** | Intake Agent |
| 강사 성과/피로 자동 분석 | **없음** | Staff Agent |
| 운영 데이터 자산화 (Palantir 컨셉) | **없음** | 핵심 차별점 |
| 법규/규제 자동 검토 | **없음** | Compliance Agent |

| 한국형 스킬 생태계 (법규·세무·노무·환불) | **없음** | k-skill Registry |

| 엑셀 import/export (기존 데이터 활용) | **없음** | 데이터 브릿지 |

### Competitive Moat

1. **오케스트레이션 구조**: 단일 도구 vs AI 팀 — 구조적 차이
2. **k-skill 생태계**: 에이전트는 범용, 스킬은 한국형. 이미 존재하는 한국 MCP 생태계(`korean-law-mcp`, `aligo-sms-mcp`, `kakao-bot-mcp`, `@portone/mcp-server`, `py-mcp-naver` 등 13개+)를 교육 맥락으로 엮는 오케스트레이터. **ChatGPT/노션으로는 이 패키징이 불가능**
3. **데이터 네트워크 효과**: 케이스가 쌓일수록 AI 판단 정확도 상승 + 엑셀 import/export로 기존 데이터 즉시 활용
4. **오픈소스**: 교육 기관의 데이터 주권 보장 → 신뢰 기반 확보
5. **도메인 팩 확장**: 보습 → IT → 태권도 → 운전면허 → 공교육 (스킬 추가만으로 가능)

## 진입 전략 vs 대체 전략

HagentOS는 기존 도구를 **대체하지 않고 위에 얹는다**.

```
기존:  클래스업(출결) + 카카오톡 + 엑셀 + 머릿속

도입:  클래스업(출결) + 카카오톡 → HagentOS 연동
       → 민원·이탈·강사·규제를 HagentOS가 처리
       → 원장은 대시보드에서 승인

장기:  HagentOS가 데이터 허브가 되면서 기존 도구 통합
```

**장점**: 기존 습관 유지 → 전환 비용 최소화 → 도입 저항 낮음
