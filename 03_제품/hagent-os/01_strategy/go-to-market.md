# Go-to-Market

> 시장 진입 전략과 확장 경로

---

## MVP 타겟

| 항목 | 내용 |
|------|------|
| **첫 사용자** | 보습/IT 학원 원장 (10~100명 규모) |
| **첫 문제** | 비정형 민원 처리 + 이탈 위험 감지 |
| **첫 도메인** | 보습/입시, IT/코딩 학원 |
| **제외** | 태권도/예체능(차량 이슈), 운전면허(규제 복잡), 대형 프랜차이즈(자체 시스템) |

### 왜 이 타겟인가

1. **의사결정 단독**: 원장 1명이 도입 결정 → 판매 사이클 최단
2. **Pain 빈도 최고**: 매일 민원 2-3건 + 이탈 위험 상시 존재
3. **기술 수용도 높음**: AX 트렌드 → AI 도입에 적극적
4. **KEG 심사 어필**: 주최사가 IT 학원 → 공감대 형성

## 진입 전략

### Phase 1: 대회 (지금 ~ 4/13)

- MVP 데모: 오케스트레이터 + Complaint Agent + Retention Agent + 통합 스케줄러
- **k-skill 레지스트리 UI + 실제 동작 스킬 5~8개** (자체 빌드 + 외부 MCP 연동)
- **외부 MCP 라이브 연동**: `korean-law-mcp`(학원법 조회), Google Calendar MCP(일정 동기화)
- Mock 데이터 기반 2분 시나리오 (스킬 장착 + MCP 연동 장면 포함)
- 오픈소스 배포 준비

### Phase 2: 초기 사용자 (대회 후 ~ 3개월)

- 대회 노출 → 관심 학원 원장 수집
- 무료 베타: 보습/IT 학원 10~30곳
- 피드백 루프: 주 1회 원장 인터뷰
- 핵심 지표: 일일 사용 시간, 승인 오버라이드율

### Phase 3: 도메인 확장 (3~6개월)

- 태권도/예체능 도메인 팩 (Vehicle Agent, Safety Agent)
- 강사 모드 추가 (Staff Agent 강화)
- 공교육 교사 파일럿 (담임 모드)

### Phase 4: 플랫폼화 (6~12개월)

- **k-skill 커뮤니티 마켓플레이스** — 원장·개발자가 스킬을 만들어 공유
- 다지점 프랜차이즈 지원
- 데이터 인사이트 대시보드 (벤치마크)
- 외부 MCP 연동 확대 (korean-law-mcp, 공공데이터API, HWP 등)

## 확장 경로

```
보습/IT 학원 (MVP)
    │
    ├── 태권도/예체능 학원 (도메인 팩)
    │
    ├── 운전면허 학원 (규제 특화 팩)
    │
    ├── 피트니스/PT (유사 구조)
    │
    └── 공교육 담임교사 (모드 전환)
         │
         └── 학교 단위 도입 (B2G)
```

## 가격 전략 (비전)

| 티어 | 대상 | 가격 | 포함 |
|------|------|------|------|
| **Free** | 개인 원장 | 무료 | 오케스트레이터 + 에이전트 2개 + 월 100건 |
| **Pro** | 중형 학원 | 월 5~10만원 | 에이전트 무제한 + 월 1,000건 + 도메인 팩 |
| **Enterprise** | 프랜차이즈/학교 | 문의 | 다지점 + 커스텀 에이전트 + 온프레미스 |

> 대회 단계에서는 가격이 아니라 **"이 구조가 작동한다"**를 증명하는 게 목표.
