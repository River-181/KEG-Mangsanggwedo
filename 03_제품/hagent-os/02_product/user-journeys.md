# User Journeys

> Before/After 흐름 + 핵심 사용 시나리오

---

## Journey 1: 학원 원장 — 김원장의 아침 (MVP)

### Before (현재)

```
07:30  출근. 어젯밤 카카오톡 확인.
       → 민원 3건 (성적불만, 행동지적, 환불요청)
       → 각각 맥락 떠올리며 답장 작성 (30분)

08:00  출결 확인. 엑셀 열어서 이상 패턴 찾기 (15분)
       → "박민준 이번달 결석 4회... 왜지?"
       → 카톡으로 학부모 연락 (10분)

08:30  강사 A 카톡: "내일 개인 사정으로 못 갑니다"
       → 대체 강사 찾기 + 학부모 안내 (20분)

09:00  신규 문의 전화 3건 직접 응대 (30분)

09:30  교육청 공문 도착. 해석 못하겠음 → 무시

10:00  드디어 수업 준비 시작
       → 이미 2.5시간 행정에 사용. 월 300만원 기회비용.
```

### After (HagentOS)

```
07:30  출근. HagentOS 대시보드 열기.

       ┌─────────────────────────────────────────┐
       │  📋 오늘의 브리핑 (Heartbeat 자동 생성)       │
       │                                         │
       │  🔴 민원 3건 — 응답 초안 생성 완료 (승인 대기)   │
       │  🟡 이탈 위험 — 박민준 (결석 4회, 환불 문의)    │
       │  🟠 스케줄 — 강사 A 내일 공결 (대체 추천 완료)  │
       │  🔵 법정 기한 — 소방 점검 D-7               │
       │  🟢 상담 일정 — 오늘 14:00 홍길동 학부모       │
       │                                         │
       │  [민원]  [이탈]  [스케줄]  [캘린더]          │
       └─────────────────────────────────────────┘

07:32  민원 카드 3장 확인
       → 홍길동 어머니(성적불만): 초안 읽기 → [전송] 클릭
       → 김서준 어머니(행동지적): 초안 수정 → [전송] 클릭
       → 박민준 아버지(환불요청): [직접 통화 필요] 표시

07:35  이탈 위험 카드 확인
       → 박민준: 근거 3개 (결석+환불+미응답)
       → [상담 일정 생성] 클릭 → 구글 캘린더 자동 동기화

07:36  스케줄 확인
       → 강사 A 내일 공결 → Staff Agent가 강사 B 추천 (가용 시간 확인 완료)
       → [대체 확정] 클릭 → 학부모 안내 메시지 자동 발송

07:38  완료. 커피 마시면서 수업 준비 시작.
       → 기존: 2시간 → HagentOS: 5분
```

## Journey 2: 담임 교사 — 이선생의 저녁 (확장)

### Before (현재)

```
18:00  퇴근... 하고 싶지만.
       → 밤 10시 카톡: "목소리 커서 잠 못 잔다"
       → 감정 소모 + 답장 고민 30분 + 작성 20분

주말   생기부 작성
       → 30명 × 1~2시간 = 학기말 30~60시간
       → 주말·공휴일 처리 45.6%

매주   행정업무 6~8시간
       → 공문 처리, 가정통신문, 보고서
       → OECD 1위
```

### After (HagentOS)

```
18:00  퇴근.

밤 10시  민원 도착 → HagentOS Complaint Agent가 자동 접수
         → 분류: [수업방식 요청] / 긴급도: [낮음]
         → 응답 초안 생성 → 다음 근무일 아침까지 대기

다음날 아침  대시보드 확인
           → "어젯밤 민원 1건 초안 완료. 검토 부탁"
           → 초안 확인 → [전송] 클릭 (2분)

학기말   생기부 초안
        → Compliance Agent가 누가기록 메모 → 행동특성 초안 생성
        → 30명 × 20초 검토 = 10분 완료
```

## Journey 3: 강사 — 박강사의 수업 후 (확장)

### Before

```
22:00  수업 끝. 학생 20명 진도 정리 (수동 메모)
       → 학부모 피드백 작성 (내일로 미룸)
       → 원장에게 보고 (카톡으로 대충)
       → 번아웃 축적
```

### After (HagentOS)

```
22:00  수업 끝.
       → Teaching Agent가 오늘 수업 요약 자동 생성
       → 학생별 피드백 초안 → 확인 후 발송
       → 원장 보고서 자동 생성
       → "오늘 수업 잘 끝남" → 퇴근
```

## Journey 0: 첫 진입 — 온보딩

```
Step 1: 기관 생성 (1분)
  → 기관명: "탄자니아 영어학원"
  → 유형: 영어
  → 규모: 소규모 (2-5명)
  → 목표: [등록 전환] [행정 자동화]
  → 문제: "민원 처리에 시간을 너무 많이 쓴다"

Step 2: 추천 AI 팀 (30초)
  → 시스템 제안:
     "당신의 기관에 맞는 팀을 추천합니다"
     1. 상담실장 (Intake Agent)
     2. 민원 담당 (Complaint Agent)
     3. 운영 매니저 (Retention Agent)

  → 원장 선택:
     [이 추천으로 시작하기]  ← 클릭
     [내가 직접 팀 구성하기]

Step 3: 첫 업무 (1분)
  → "어떤 업무부터 시작할까요?"
  → [Mock 데이터로 체험하기]  ← MVP에서는 이것
  → [실제 데이터 연결하기]    ← v2

Step 4: 대시보드
  → Mock 민원 3건 로드
  → 에이전트 실행 → 결과 카드 표시
  → 원장: "오, 이런 거구나" → 승인 체험
```
