# NotebookLM CLI 운영 가이드

## 구조

```text
_system/tools/
├── pyproject.toml
├── .env.example
├── bootstrap.sh
├── nlm/
│   ├── nlm.md
│   └── 260406-nlm-운영가이드.md
└── runtime/
    └── nlm/
        ├── profiles/
        └── chrome-profiles/
```

## 설치 기준

| 항목 | 값 |
|---|---|
| 패키지 | `notebooklm-mcp-cli==0.5.5` |
| 설치 방식 | `uv sync --project _system/tools` |
| 실행 명령 | `uv run --project _system/tools nlm <command>` |
| 인증 | `nlm login` 세션 기반 |

## 기본 환경

```bash
cp _system/tools/.env.example _system/tools/.env
source _system/tools/.env
uv sync --project _system/tools
uv run --project _system/tools nlm login
```

## 권장 워크플로우

1. `notebook create`
2. `research start --mode deep --notebook-id`
3. `research status`
4. `research import`
5. `report create` 또는 `mindmap create`
6. 결과를 note로 정리
7. `[[master-evidence-ledger]]` 기록

## 런타임 데이터 규칙

- `runtime/nlm/profiles/` — 계정 세션
- `runtime/nlm/chrome-profiles/` — 브라우저 자동화 데이터
- 이 경로는 Git에 올리지 않는다.

## 주의

- 세션 만료가 짧을 수 있다.
- 공용 저장소에는 쿠키/프로필/토큰을 절대 올리지 않는다.
