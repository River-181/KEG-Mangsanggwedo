# 온보딩 가이드

> 새 팀원이나 AI 에이전트가 이 워크스페이스에 처음 진입할 때 읽는 문서.

## 이 프로젝트는 무엇인가

2026 제1회 KEG 바이브코딩 콘테스트 참가 워크스페이스. 2인 팀 + AI 에이전트가 7일간 교육 분야 AI 솔루션 MVP를 개발한다.

## 읽는 순서

1. **[[바이브코딩공모전_공지]]** — 대회 완전 개요서 (대회의 모든 것)
2. **[[00 HOME]]** — 워크스페이스 전체 맵
3. **`.agent/AGENTS.md`** — AI 에이전트 역할과 규칙
4. **[[team_profiles]]** — 팀 구성

## 디렉토리 구조

```
01_대회정보/  — 대회 규칙, 일정, 팀 프로필
02_전략/      — 전략, 분석, 의사결정 기록
03_제품/      — 제품 설계, 스펙, 아키텍처
04_증빙/      — AI 사용 로그, 의사결정 기록, 프롬프트 카탈로그
05_제출/      — AI 리포트, 체크리스트, 회고
assets/       — PDF, DOCX, 스크린샷, 데모 영상
app/          — 실제 제출 제품
tests/        — 테스트 파일
.claude/      — Claude Code 전용 설정, 에이전트 정의
.agent/       — 에이전트 공용 설정 (Cursor/Copilot 등)
.context/     — 공유 맥락 (메모리, 플러그인, MCP)
```

## 핵심 규칙

1. 모든 마크다운은 **옵시디언 마크다운** — 위키링크 `[[]]` 사용
2. 모든 노트에 **YAML frontmatter** 포함 (최소 `tags`, `date`)
3. 작업 완료 시 `04_증빙/ai-usage-log.md`에 **1줄 로그** 남기기
4. `assets/originals/`의 파일은 **수정 금지** (원본 보관용)
5. 제출 마감: **2026-04-13 24:00** — 이후 커밋은 부정행위
