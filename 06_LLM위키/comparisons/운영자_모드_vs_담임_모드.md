# 운영자 모드 vs 담임 모드

## Snapshot

| 축 | 운영자 모드 | 담임 모드 |
|---|---|---|
| 현장 마찰 선명도 | 높음 | 높음 |
| KEG/학원 맥락 직접성 | 매우 높음 | 중간 |
| 제도/규정 복잡도 | 중간 | 높음 |
| 데모 전달력 | 높음 | 매우 높음 |
| 데이터 범위 | 넓음 | 중간 |
| 첫 MVP 범위 위험 | 높음 | 중간 |

## Current Read

- 바텀업 학원 리서치 쪽은 운영자 모드가 더 자연스럽다고 본다.
- 다만 데모 장면의 선명도는 담임 모드가 강하다.
- 그래서 제품 코어는 운영자 모드, 데모 확장 컷은 담임 모드로 잡는 전략이 가능하다.

## Decision Pressure

- 첫 사용자를 원장/실장으로 볼지, 담임/강사로 볼지에 따라 데이터 모델과 approval flow가 달라진다.
- 이 비교는 `problem-scorecard`와 직접 연결돼야 한다.
