# 학원 운영 데이터 자산화 우선순위

## Summary First

학원에서 먼저 자산화할 데이터는 성적보다 `민원/상담/재등록/예외 승인`이며, 이유는 이 데이터가 운영 판단과 실수 방지에 직접 연결되기 때문이다.

## Priority Stack

1. 민원/상담 로그 표준화
2. 재등록/이탈 이벤트 정의
3. 학생-학부모-강사-반 연결 그래프
4. 수납/환불/할인 승인 로그
5. 법규/점검/증빙 타임라인

## Interpretation

- 운영 AI의 품질은 모델 능력보다 `누적 맥락의 정확도`에서 갈린다.
- 따라서 초기 제품은 입력 기능을 많이 만드는 것보다, 어떤 로그를 구조화할지를 먼저 정해야 한다.
