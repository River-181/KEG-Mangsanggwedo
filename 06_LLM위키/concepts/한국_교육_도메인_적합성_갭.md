# 한국 교육 도메인 적합성 갭

## Summary First

Paperclip은 범용 business orchestration에 가깝고, 우리 문제는 한국 교육 운영이다. 따라서 translation unit은 기능 복제가 아니라 `도메인 문맥`, `관계 구조`, `규정/증빙`, `운영 책임`이다.

## What Can Be Borrowed

- control plane 시각
- org chart / governance / approval 흐름
- runtime + control plane 이원 구조
- budget / task trace 개념

## What Must Be Transformed

- `company operations` -> `academy/class operations`
- `mission growth` -> `운영 안정 + 교육 품질 + 유지율 + 분쟁 관리`
- template -> `학원 유형별 운영 pack` 또는 `k-skill bundle`
- multi-company -> `지점 운영` 또는 `원장/실장/강사 다중 역할`

## Korea-specific Domains

### 학부모 민원

- 감정 노동
- 야간 연락
- 반복 질문
- 증빙 보존

### 재등록 / 이탈 방어

- 매출과 직접 연결
- 출결, 성과 설명, 상담 이력, 시험 일정과 연결

### 강사 스케줄과 workload

- 강사 이탈이 반 운영 품질과 환불/민원까지 연결
- 보강, 연속성, 피로도와 함께 봐야 함

### 법규 / 세무 / 차량 / 시설

- 학원 유형별로 운영 OS가 다르게 생김
- 공통 코어 + domain pack 구조가 자연스러움

## Biggest Fit Gaps

- 한국 학원은 학부모/강사 관계가 제품 UX에 직접 반영돼야 한다.
- 공교육은 compliance-aware system이 필요하다.
- 사교육은 operations OS로 보여야 한다.
- `goal` 중심보다 `운영 case` 중심 구조가 더 자연스러울 수 있다.
- `cost control`은 민원 비용, 환불 리스크, 이탈 비용까지 넓어진다.

## Not Suitable

- `zero-human` 서사를 그대로 가져오는 것
- 미국 SaaS 회사 기준 taxonomy
- 도메인 차이를 무시한 universal workflow
