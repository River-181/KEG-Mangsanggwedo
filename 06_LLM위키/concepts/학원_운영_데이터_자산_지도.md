# 학원 운영 데이터 자산 지도

## Summary First

학원 운영 데이터는 이미 많다. 문제는 흩어져 있고 `케이스`, `승인`, `후속 행동`, `성과`와 연결되지 않아 자산이 되지 못한다는 점이다.

## Asset Categories

### 학생 데이터

- 기본 인적 정보, 학년, 반, 등록/휴원/퇴원 이력
- 출결 패턴, 보강 이력
- 성적/평가/오답/학습 진도
- 상담 메모, 행동 특이사항, 이탈 징후

### 학부모 데이터

- 문의/민원/상담 채널과 이력
- 민감 포인트, 약속 이력
- 결제 패턴, 환불/할인 요청, 재등록 신호

### 강사 데이터

- 담당 반, 시간표, 대타/휴강 이력
- 수업 품질 피드백, 민원 연관도, 유지율
- 계약 형태, 정산, 업무량, 피로 신호

### 운영/재무 데이터

- 수납, 미수금, 환불, 할인, 매출 추이
- 모집 채널, 상담 전환율, 등록률, 재등록률, 이탈률
- 강의실/차량/시설 가동률
- 세무 증빙, 급여/정산, 비용 구조

### 케이스 데이터

- 민원 시점, 주제, 심각도, 담당자, 처리 결과
- 환불/예외 승인 기록과 근거
- 재등록 방어 시도와 결과
- 해결 시간과 재발 여부

## Why It Matters

- 강한 차별점은 단일 LLM 응답이 아니라 `누적 맥락`과 `예측 가능한 운영 판단`이다.
- 가장 먼저 자산화할 데이터는 성적보다 `민원/상담/재등록/예외 승인`일 수 있다.
- 장기적으로는 학원별 운영 know-how를 `skill + data` 패키지로 묶을 수 있다.

## Priority Order

1. 민원/상담 로그 표준화
2. 재등록/이탈 이벤트 정의
3. 학생-학부모-강사-반 연결 그래프
4. 수납/환불/할인 승인 로그
5. 법규/점검/증빙 타임라인

## Product Implication

- 운영자가 체감하는 AI 품질은 모델 지능보다 `누적 맥락의 정확도`에 좌우된다.
- 자동 수집과 사람 확인이 필요한 데이터를 처음부터 분리해 설계해야 한다.
