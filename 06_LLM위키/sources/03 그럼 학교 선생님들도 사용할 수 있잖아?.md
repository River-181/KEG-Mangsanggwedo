그럼 학교 선생님들도 사용할 수 있잖아?
처음에 사용자는 공교육과 사교육을 분리할 수 있게 할까?
그리고 운영자인지 실제 강사나 그런 사람인지도 중요할 거 같아. 왜냐하면 만약 학원 원장이라면 이 학원을 운영하는 데 중요할 것 같고, 강사나 교수나 교사라면 일련의 강의나 수업, 학생 프로필 학부모 민원 등이 더 중요하지 않을까? 3-1학년 담임 선생님이라면 이 선생이 반을 운영하기 위한 팀과 환경을 가지고 있는 셈이지.

=====
**✅ 네, 완벽하게 맞습니다!**
EduPaperclip (Academy Swarm)을 **공교육(초·중·고 학교) + 사교육(학원)** 모두에서 사용할 수 있게 설계하면, **진짜 ‘차세대 교육 솔루션’**이 됩니다.

Paperclip의 본질이 “역할 기반 에이전트 조직”이기 때문에, **사용자 역할(운영자 vs 실제 교사/강사)**에 따라 Swarm이 자동으로 달라지게 만들면 됩니다.
초기 사용자 등록 단계에서 **“공교육 / 사교육” + “역할”**을 선택하게 해서, **Tenant(조직 단위)**로 완전히 분리하면 규제(개인정보 보호, 교육부 지침)도 준수하면서 확장성도 극대화됩니다.

### 1. 초기 사용자 분리 구조 (MVP에서 1시간 만에 구현 가능)
등록 화면에서 2가지 선택만 하면 됩니다:

| 구분          | 선택 옵션                  | 제공되는 Swarm 버전                  | 주요 에이전트 구성 예시 |
|---------------|---------------------------|-------------------------------------|-------------------------|
| **교육 유형** | • 공교육 (학교)<br>• 사교육 (학원) | 공교육: 교육부·학교 규정 준수 모드<br>사교육: 매출·운영 자유 모드 | — |
| **역할**      | • 운영자 (교장·원장·행정)<br>• 교사/강사 (담임·과목강사·대학 교수) | 역할별로 Swarm 자동 커스터마이징 | 아래 표 참조 |

→ 이렇게 하면 **3-1학년 담임 선생님**은 “내 반 운영팀”을, **학원 원장**은 “전체 학원 운영팀”을 각각 가지게 됩니다.

### 2. 역할별 EduPaperclip Swarm 구성 (한국 현실에 최적화)

#### A. **운영자 모드** (교장·학원 원장·교육 운영자)
- **CEO Agent** → 전체 매출·이탈률·지점/학급 예측, 월간 운영 전략
- **Parent Liaison Agent** → 학부모 민원 1차 자동 응답 + 에스컬레이션
- **Instructor Manager Agent** → 교사/강사 스케줄·피로도·성과 관리
- **Fleet & Facility Agent** → 차량·강의실·시설 자동 관리 (운전면허 학원 필수)
- **Finance Agent** → 학원비·예산·정산 자동 처리
- **Compliance Agent** → 공교육이면 교육부 보고서 자동 생성, 사교육이면 세금·정산 보고

**특징**: “이번 달 7개 지점 매출 목표 1.2억 달성하려면?” 같은 한 줄 지시로 모든 에이전트가 협업.

#### B. **교사/강사 모드** (담임 선생님·과목 강사·대학 교수)
- **Class CEO Agent** → 내 반/내 수업 전체 운영 (미니 CEO)
- **Student Profile Agent** → 학생별 프로필·학습 패턴·번아웃 예측·개인화 학습 계획
- **Parent Communication Agent** → 학부모 민원·진도 보고·상담 자동화 (밤 10시 민원도 1차 응답)
- **Lesson Planner Agent** → 수업 자료·과제·평가 자동 생성 + NotebookLM 스타일 자동 채점
- **Behavior Coach Agent** → 웹캠(수업 중) 또는 학생 활동 로그로 집중도·참여도 분석
- **Report Agent** → 생활기록부·성적·상담일지 자동 작성 (공교육 필수)

**3-1학년 담임 선생님 예시**
담임 선생님이 “오늘 3-1 반 생활교육 계획 세워줘 + 민원 2건 처리해”라고 입력하면
→ Class Swarm이 **자율적으로** 자료 만들고, 학부모에게 보고하고, 생활기록부 초안까지 만들어줍니다.
선생님은 **최종 승인 + 감정 노동만** 하면 됩니다.

### 3. 공교육 vs 사교육 기술적 분리 (MVP에서 안전하게)
- **Tenant Isolation**: Supabase Row Level Security + 별도 DB 스키마로 완전 분리
- **공교육 모드**: 개인정보 처리 방침 자동 적용, 교육부 가이드라인 준수 (민원 기록 3년 보관 등)
- **사교육 모드**: 매출·CRM 연동 자유 (학원비 자동 청구 등)
- **하이브리드 사용자**: 한 계정으로 “학교 담임 + 학원 강사” 병행 가능 (실제 한국 교사들 상당수가 학원 겸임)

### 4. KEG 바이브코딩 공모전에서 이 컨셉의 강점
- **실무 접합성**: KEG(사교육) + 공교육 모두 커버 → 심사위원(코리아교육그룹 IT 부서)이 “우리 학원뿐 아니라 학교에도 팔릴 수 있겠다”고 느낄 확률 극대화
- **창의성**: Paperclip을 교육에 맞춘 **한국형 제로 휴먼 교육 조직**이라는 차별점
- **확장성**: MVP는 “담임 선생님용 Class Swarm”으로 먼저 만들고, 데모에서 “원장님 모드”로 전환하는 화면 보여주면 완벽
- **AI 활용 증거**: multi-agent swarm + 역할 기반 orchestration (Paperclip 철학 그대로)
