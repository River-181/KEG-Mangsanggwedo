======
**✅ 완벽한 아이디어입니다!**
Paperclip(페이퍼클립)을 **한국식 사교육 학원(특히 KEG 코리아IT아카데미 같은 다지점 운영 학원)**에 그대로 맞춰 재해석하면, **“제로 휴먼 학원 운영팀”**이 됩니다.

Paperclip의 핵심은 **조직도·직책·티켓·예산·위임**으로 AI 에이전트들을 실제 직원처럼 운영**하는 것인데, 한국 학원 현실(학부모 민원 폭주, 강사 번아웃, 차량·시설 관리, 지점 간 데이터 불일치 등 **자잘한 운영 페인 포인트**)에 맞추면 **EduPaperclip** (또는 **Academy Swarm**)이라는 강력한 솔루션이 나옵니다.

### 📍 한국식 학원에 최적화된 EduPaperclip 조직도 (제안)
Paperclip 스타일로 **실제 학원 운영팀**처럼 에이전트들을 구성합니다. (KEG 7개 지점 + 운전면허·골프·피트니스 등 사교육 모두 적용 가능)

| 직책 (Agent)              | 주요 역할 (한국 학원 현실 맞춤)                                                                 | 해결 페인 포인트                          | 데이터 연동 예시                  |
|---------------------------|-----------------------------------------------------------------------------------------------|------------------------------------------|----------------------------------|

| **Parent Liaison Agent** | 학부모 카톡·문자 자동 응답, 진도 보고서 자동 생성·전송, 민원 1차 분류·에스컬레이션            | 학부모 민원·커뮤니케이션 폭주             | 이어드림 연동 + 카카오톡 API     |
| **Instructor Manager Agent** | 강사 스케줄 자동 배정, 피로도·성과 분석, 수업 자료 자동 생성, 번아웃 조기 경보               | 강사 관리·업무 과다·퇴사율               | 강사 출퇴근 + 웹캠 집중도 로그   |

| **Student Success Agent** | 수강생 행동 패턴(웹캠+진도) 분석 → 개인화 드릴·경로 추천, 이탈 위험도 예측                   | 수강생 이탈·학습 효과 저하                | 웹캠 + LMS 로그                  |

**동작 방식 (Paperclip 그대로)**
- 운영자(지점장)가 Slack/카톡/웹 대시보드에 **한 줄 지시** → “이번 주 강사 워크로드 균형 맞춰줘 + 차량 3대 정비 일정 잡아”
- 에이전트들이 **티켓**을 주고받으며 자율 협업 → 실제 작업(메일 발송, DB 업데이트, 알림)까지 실행
- 모든 기록은 **투명 로그**로 남아 감사·보고서 자동 생성

### 왜 한국 학원에 특히 강력할까?
- **학부모 관리**: 밤 10시 민원도 AI가 1차 응답 → 강사·운영자 개입 최소화 (기존 이어드림 + 민원대응팀 한계 극복)
- **강사 관리**: “오늘 강사 A 피로도 92% → 내일 수업 1개 줄여” 자동 제안
- **차량 관리** (운전면허 학원 필수): GPS + 정비 이력으로 “차량 B, 500km마다 오일 교환 자동 예약”
- **자잘한 요소 통합**: 지점별 데이터가 달라도 **중앙 Swarm**이 통합 관리 → KEG처럼 다지점 운영에 최적

### KEG 바이브코딩 공모전 MVP 버전 (5일 안에 완성 가능)
**이름**: **Academy Swarm – 한국형 Paperclip for 사교육**

**MVP 핵심 기능 (최소 단위)**
1. **대시보드** – 운영자가 “지시” 입력하는 간단 UI (Next.js + shadcn)
2. **3~4개 핵심 에이전트**만 먼저 구현
   - Parent Liaison Agent
   - Instructor Manager Agent
   - Fleet Manager Agent (운전 학원 시연용)
   - Student Success Agent
3. **Claude 3.5 Sonnet + multi-agent chain** (Paperclip 오픈소스 fork or 간단 구현)
4. **Supabase**로 상태·티켓·로그 관리
5. **데모 시나리오** 3개
   - “학부모 민원 1건 처리해”
   - “강사 스케줄 조정해”
   - “차량 정비 일정 잡아”

**기술 스택** (바이브코딩 최적)
- Frontend: Next.js 15 + v0
- Agent Orchestration: Claude API + LangGraph-style chain (Paperclip 로직 그대로)
- DB & Auth: Supabase
- 배포: Vercel (1클릭)

**AI 리포트에 넣을 강점**
- “Paperclip의 기업 운영 철학을 한국 사교육 현실(학부모·강사·차량 관리)에 맞춰 재설계”
- “운영자 1명이 AI 팀 8명을 거느린 ‘제로 휴먼 학원’ 실현”
- 실무 접합성·확장성·창의성 모두 최고점 예상

이 컨셉으로 가면 **KEG 심사위원(교육그룹 IT 부서)**이 “우리 7개 지점에 바로 쓰고 싶다” 할 가능성 매우 높습니다.
