> 플레이북 §2 "주제 공개 직후 6~12시간 의사결정 스프린트" 산출물

## 1. 문제 요약

**한 문장**: 한국 학원 운영자와 담임 교사는 행정·민원·관리 잡무에 하루 3~8시간을 빼앗겨, 교육이라는 본업에 집중하지 못한다.

**근거** (7개 소스 교차 확인):
- 원장: 하루 3시간 행정 = **월 300만원 기회비용** (bati.ai)
- 교사: 주당 행정 6~8시간 = **OECD 1위** (TALIS 2024)
- 교사: 주당 생기부 5~8시간 = 주말·공휴일 처리 45.6%
- 교사: 학부모 민원 스트레스 **56.9% = 세계 2위** (TALIS 2024)
- 교사: 악성 민원 경험 **46.8%** (교사노조연맹, n=4,068)
- 원장: 출결·수납·알림은 자동화됐지만 민원·이탈·강사 관리는 **100% 수동**
- 학령인구 2026년 **500만 붕괴** → 생존 위기 = 솔루션 수요 급상승

**경쟁 공백** (화이트스페이스):
- 기존 솔루션(클래스업, 온하이, LGU+ 슈퍼스쿨)은 루틴만 처리
- 역할 기반 AI 에이전트 오케스트레이션: **경쟁사 없음**
- Palantir처럼 교육 운영 데이터를 완전히 자산화하는 구조: **없음**

## 2. 사용자 페르소나 후보

> ⚠️ 아직 확정 전. 다음 작업으로 최종 선택.

### 후보 A — 학원 원장 (사교육 운영자)

| 항목       | 내용                                       |     |
| -------- | ---------------------------------------- | --- |
| 이름       | 김원장, 45세, 보습학원 원장 10년차                   |     |
| 핵심 상황    | 강사 3명, 학생 60명. 직접 수업도 하면서 민원·상담·이탈 관리까지  |     |
| 가장 아픈 순간 | 밤 11시 학부모 카톡 민원. 강사 갑자기 그만두겠다는 연락        |     |
| 지금 해결책   | 카카오톡 + 클래스업(출결) + 엑셀 + 머리 속              |     |
| 원하는 것    | "내가 수업에 집중할 수 있게 나머지를 누군가 대신 처리해줬으면"     |     |
| 데이터 자산화  | 차량 기록, 강사 프로필, 상담 이력, 매출/재등록률 → 맥락 통합 관리 |     |

**지지 근거**: KEG 주최사 공감 · AX 조직 트렌드 · 노란봉투법 직후 타이밍 · 확장성 5점

### 후보 B — 담임 교사 (공교육 실무자)

| 항목 | 내용 |
|------|------|
| 이름 | 이선생, 32세, 중학교 담임 5년차 |
| 핵심 상황 | 30명 학생. 수업·생기부·민원·행정 모두 혼자 |
| 가장 아픈 순간 | "수업 중 목소리가 커서 아이가 잠 못 잔다" 밤 10시 카톡 |
| 지금 해결책 | NEIS + 카카오톡 + 개인 메모 + 초과근무 |
| 원하는 것 | "수업 외 업무가 완전히 사라졌으면" |
| DMZ 역할 | 교사와 학부모 사이 AI 완충층 (이승보 아이디어) |

**지지 근거**: OECD TALIS 최강 수치 · 데모 직관성 최고 · NEIS 연동 불필요로 7일 구현 용이

## 3. 해결 전 / 후 흐름

### Before (현재)

```
[원장] 오전
  → 어젯밤 민원 3건 직접 답장 (30분)
  → 출결 이상 학생 파악 (엑셀 확인, 15분)
  → 강사 스케줄 조정 (카톡, 20분)
  → 신규 문의 전화 응대 (직접, 30분)
  → 교육청 공문 해석 (모름 → 무시)
  합계: ~2시간 행정 → 월 300만원 기회비용

[교사] 퇴근 후
  → 밤 11시 민원 카톡 직접 답장
  → 주말 생기부 30명 × 1~2시간
  → 수업 계획 야간 작업
  합계: 주 11~16시간 비교육 업무
```

### After (EduPaperclip)

```
[원장/교사] 아침 5분
  → 대시보드 자동 브리핑:
     "어젯밤 민원 2건 처리완료 (승인 대기)
      이탈 위험 학생 1명 감지
      강사 A 성과 하락 알림"
  → 원클릭 승인 3번 → 완료

특이사항 없는 날: 완전 자동 (Level 0)
복잡한 민원: 초안 → 편집 후 발송 (Level 1)
법적·감성 판단: AI 정보 제공 → 사람 직접 (Level 4)
```

## 4. MVP 범위 / 제외 범위

### ✅ MVP 포함 (7일 안에 데모 가능)

| 기능 | 에이전트 | 제로휴먼 레벨 | 근거 |
|------|---------|:------------:|------|
| 학부모 민원 분류 + 응답 초안 | Complaint Agent | 1 | TALIS 56.9%, Grok 레전드 민원 |
| 재원생 이탈 징후 감지 | Retention Agent | 0~1 | bati.ai 월 300만원 |
| 원장/교사 한 줄 → 에이전트 분산 실행 | Orchestrator | — | Paperclip 핵심 개념 |
| 에이전트 결과물 승인 대시보드 | UI | — | Level 1 구현 |
| 데이터 자산화 기반 (DB 구조) | Infrastructure | — | Palantir 컨셉 기반 |

### ❌ MVP 제외 (범위 초과 or 레드오션)

| 기능 | 제외 이유 |
|------|-----------|
| 출결·수납·알림 자동화 | 클래스업·온하이 이미 커버. 레드오션 |
| 생기부 단독 | LGU+ 슈퍼스쿨·왓퀴즈 이미 있음 |
| 차량 관리 단독 | IoT 없이 데모 설득력 없음 |
| NEIS 직접 연동 | 개인정보 규제. Mock 데이터로 대체 |
| 카카오 채널 연동 | 7일 내 구현 난이도 높음 |
| 강사 성과 대시보드 전체 | v2로 |
| 공문·규제 자동 브리핑 | v2로 |
