## Step 2 — NotebookLM 실행 프롬프트 (순서대로)

### 프롬프트 1: 핵심 수치 교차 검증
```
업로드된 9개 소스에서 공교육 교사와 사교육 학원 원장의 업무 부담을
보여주는 정량 수치를 모두 추출하고, 소스별로 교차 검증해서 일치/불일치를 표로 정리해줘.
특히 행정업무 시간, 민원 스트레스 비율, 재정적 손실 수치에 집중해.
```

### 프롬프트 2: 화이트스페이스 확인
```
업로드된 소스들에서 기존 에듀테크/학원 관리 솔루션(클래스업, 온하이,
LGU+ U+슈퍼스쿨, 왓퀴즈 등)이 커버하지 못하는 영역을 추출해줘.
AI 에이전트 오케스트레이션이 필요한 이유도 정리해줘.
```

### 프롬프트 3: 현장 목소리 수집
```
Grok 파일(X 현장 목소리)에서 학원 원장과 담임 교사가 가장 많이 호소하는
페인포인트를 역할별로 각 5개씩 추출해줘.
실제 발언(인용구)을 포함하고, 어느 소스에서 나왔는지 표시해줘.
```

### 프롬프트 4: MVP 후보 비교
```
P-001 학원 원장 운영 보조 Swarm과 P-002 담임교사 반 운영 Swarm 중
어느 쪽을 7일 MVP로 먼저 만들어야 할지 결정하는 데 필요한
근거들을 모든 소스에서 찾아서 표로 정리해줘.
평가 기준: 현장성, AI 필요성, 7일 구현 가능성, 데모 설득력, 확장성.
```

### 프롬프트 5: 최종 합성 — FAQ 생성
```
업로드된 소스 전체를 기반으로 EduSwarm 프로젝트의 핵심 질문 3개에 답해줘:

1. 지금 당장 가장 아픈 사람은 누구인가? (공교육 교사 vs 사교육 원장)
2. 기존 솔루션에서 AI swarm이 추가할 수 있는 가장 큰 차별점은 무엇인가?
3. 7일 만에 만들 수 있는 데모의 핵심 장면은 무엇인가?
```
