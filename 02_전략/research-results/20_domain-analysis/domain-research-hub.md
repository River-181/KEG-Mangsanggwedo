# 한국 학원 운영 도메인 분석 허브

> 개발 spec 전에 한국 학원 운영의 실제 업무 구조, 기존 솔루션 한계, 법적 의무, 데이터 자산화 가능 영역을 정리하는 폴더.

## 구성

- [[hagwon-operations]]: 원장/실장이 처리하는 일상 운영 업무 맵
- [[pain-points]]: 기존 솔루션 기능 범위와 현장 불만
- [[legal-requirements]]: 학원 운영자가 놓치면 안 되는 법적 의무
- [[data-assets]]: 운영 데이터를 자산으로 바꿀 수 있는 영역

## 작성 원칙

- 기능 spec으로 내려가지 않는다.
- 사실, 가설, 오픈 질문을 분리한다.
- `운영자 모드`와 `도메인팩` 판단에 도움이 되는 내용만 남긴다.
- 공식/1차 자료를 우선하고, 부족한 부분은 기존 리서치 문서를 보조 근거로 사용한다.
