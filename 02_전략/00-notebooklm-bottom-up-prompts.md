# NotebookLM Bottom-Up Research Prompts

> 한국 학원 생태계를 바닥부터 조사하기 위한 NotebookLM 전용 프롬프트 세트.
> 목적은 `기능 정의`가 아니라 `문제 정의와 페르소나의 증거`를 만드는 것이다.

## 사용 원칙

- 업로드 자료는 raw source, 기사, 리서치 결과, 메모를 섞는다.
- 각 프롬프트는 `운영 마찰`, `반복 업무`, `예외 처리`, `감정 노동`, `승인 지점`을 우선 추출하게 설계한다.
- 결과는 `problem-bank`나 `spec`으로 바로 옮기지 말고 먼저 검토한다.

## 추천 입력 자료

- `02_전략/00-bottom-up-academy-research-source.md`
- `02_전략/00-problem-definition-source.md`
- `02_전략/research-results/` 하위 raw 리서치
- `06_LLM위키/sources/` 하위 원문 메모
- 학원 관련 기사, 인터뷰, 커뮤니티 스크린샷, 제도 자료

## Prompt 1. 학원 운영자의 하루를 바닥부터 복원

```text
업로드한 자료를 바탕으로 한국 학원 운영자(원장/실장/상담 담당)의 하루 운영 흐름을 바닥부터 복원해 주세요.

중요:
- 탑다운 시장 분석이 아니라 실제 운영 흐름을 먼저 그려야 합니다.
- 기능 제안보다 반복 업무와 병목을 먼저 추출하세요.

반드시 다음 형식으로 정리하세요.
1. 하루 흐름 타임라인
2. 반복 업무 목록
3. 가장 자주 발생하는 예외 상황
4. 감정 노동이 발생하는 순간
5. 사람 승인/판단이 꼭 필요한 지점
6. 기존 솔루션이 커버하는 부분
7. 기존 솔루션이 비워 둔 부분
8. AI team 구조가 특히 필요한 부분

가능하면 `신규 문의`, `등록`, `시간표/보강`, `민원`, `재등록/이탈`, `강사 조율`, `회계/법무/차량/시설` 같은 운영 요소를 빠뜨리지 말고 정리하세요.

출력은 표와 짧은 해설을 섞어서 작성하세요.
```

## Prompt 2. 강사/담임/실무자의 운영 마찰 추출

```text
업로드한 자료에서 학원 강사, 공교육 담임, 교육 실무자가 실제로 반복적으로 겪는 운영 마찰을 추출해 주세요.

중요:
- 수업 내용보다 운영 마찰을 우선 봅니다.
- "어떤 기능이 있으면 좋다"보다 "어느 순간 막히는가"를 먼저 적습니다.

각 pain point를 다음 형식으로 정리하세요.
- 상황: 누가 어떤 순간에 무엇 때문에 막히는가
- 현재 대처 방식: 지금은 어떻게 버티는가
- 누적 비용: 시간, 감정, 누락, 재작업, 갈등 중 어떤 비용이 생기는가
- 데이터 흔적: 이 상황이 남기는 기록/메시지/문서/엑셀/메모는 무엇인가
- agent 분업 필요성: 단일 chatbot으로는 부족한 이유가 있는가

마지막에는 가장 강한 pain 10개만 남기고 우선순위를 매겨 주세요.
```

## Prompt 3. 학부모 민원과 커뮤니케이션 구조 해부

```text
업로드한 자료를 바탕으로 한국 학원과 학교에서 학부모 커뮤니케이션이 어떻게 발생하고 왜 감정 노동이 되는지 분석해 주세요.

다음에 집중하세요.
- 어떤 채널로 들어오는가
- 어떤 시간대에 많이 오는가
- 어떤 유형의 민원이 반복되는가
- 응답 전에 맥락 확인이 왜 어려운가
- 누가 답하고 누가 승인해야 하는가
- 잘못 대응했을 때 무엇이 커지는가

출력 형식:
1. 민원 유형 분류표
2. 응답 흐름도
3. 사람이 반드시 끼어야 하는 판단 지점
4. AI가 초안을 만들 수 있는 구간
5. AI가 건드리면 안 되는 구간

기능 제안은 마지막에 3개 이하로만, 아주 보수적으로 적으세요.
```

## Prompt 4. 학원 도메인별 차이 찾기

```text
업로드한 자료를 바탕으로 한국 학원 생태계를 도메인별로 비교해 주세요.

비교 대상 예시:
- 보습/입시
- IT 아카데미/부트캠프
- 태권도/예체능
- 운전면허

반드시 다음 질문에 답하세요.
1. 각 도메인에서 운영자가 실제로 가장 자주 처리하는 일은 무엇인가
2. 강사/실무자의 반복 업무는 무엇인가
3. 학부모 또는 보호자와의 접점이 어떻게 다른가
4. 수강생 관리 방식은 어떻게 다른가
5. 어떤 도메인이 `EduPaperclip`의 첫 타겟으로 가장 적합한가

출력은 비교표 + 추천 이유로 작성하세요.
중요: "시장 규모"보다 "운영 마찰의 선명도"를 우선으로 판단하세요.
```

## Prompt 5. 데이터 자산화 관점에서 다시 보기

```text
업로드한 자료를 바탕으로 한국 학원 운영에서 어떤 데이터가 실제 자산이 될 수 있는지 정리해 주세요.

중요:
- 단순 개인정보 목록이 아니라, 운영 개선과 의사결정에 실제로 쓰이는 데이터를 찾습니다.
- `Palantir`식 데이터 자산화 관점을 참고하되, 현장 업무에서 출발하세요.

각 데이터 항목마다 다음을 정리하세요.
- 어디서 생기는가
- 누가 입력하거나 남기는가
- 어떤 운영 판단에 쓰일 수 있는가
- 민감도/보안 주의점은 무엇인가
- 로컬 LLM 또는 private deployment가 필요한 이유가 있는가

마지막에는
1. MVP에 꼭 필요한 데이터 5개
2. 나중에 붙여야 할 데이터 5개
3. 지금은 버려야 할 데이터 5개
를 구분해 주세요.
```

## Prompt 6. spec 전에 확정해야 할 질문만 남기기

```text
앞선 분석 결과를 바탕으로, 지금 당장 spec으로 내려가면 위험한 이유를 정리해 주세요.

그리고 spec 전에 반드시 확정해야 하는 질문만 남겨 주세요.

질문 범주:
- 첫 사용자
- 첫 operational pain
- 첫 approval flow
- 첫 데이터 범위
- AI가 해도 되는 일 / 안 되는 일

출력 형식:
1. 지금 spec으로 내려가면 생기는 위험 5개
2. spec 전에 반드시 답해야 할 질문 7개
3. 그 질문에 답하기 위해 추가 조사해야 할 자료 5개
```

## Research Agent Output Contract

- `운영 흐름`
- `반복 pain`
- `강한 근거`
- `약한 가설`
- `AI team 필요성`
- `spec 전 미결 질문`
