# Demo Critical Path

> "시연에 필요한 최소 사용자 여정"
> 이 여정이 막히면 데모 불가. 나머지는 다 버려도 된다.

---

## 핵심 전제

- **시연 시간**: 2분
- **목표 반응**: 심사위원 "와"
- **핵심 장면**: 원장/교사 한 줄 → AI 팀 병렬 실행 → 결과물 즉시 출력
- **Mock 데이터 필수**: 민원 3건, 학생 20명 출결 데이터, 강사 3명

---

## 최소 사용자 여정 (Critical Path)

```
Step 0: 로그인 or 대시보드 진입
  ↓ [30초]
Step 1: 오늘의 AI 브리핑 자동 표시
  "어제 민원 2건 | 이탈 위험 1명 | 승인 대기 3건"
  ↓ [10초]
Step 2: 사용자가 한 줄 지시 입력
  "오늘 민원 처리하고 이번 주 이탈 위험 학생 알려줘"
  ↓ [30초 — 핵심 장면]
Step 3: 에이전트 팀 병렬 실행 (실시간 표시)
  [Complaint Agent 실행 중...] ✓ 완료
  [Retention Agent 실행 중...] ✓ 완료
  ↓ [20초]
Step 4: 결과물 카드 표시
  민원 초안 카드 / 이탈 학생 카드
  ↓ [10초]
Step 5: 원클릭 승인
  "전송" 버튼 클릭 → 완료 확인
  ↓ [10초]
Step 6: (선택) 교사 모드로 전환 장면
  "사교육 원장 → 공교육 교사 모드" 1초 전환
  → 확장성 어필
```

**총 여정: 2분 10초**

---

## Step별 구현 우선순위

| Step | 구현 필수 여부 | 없으면 어떻게 되나 |
|------|:------------:|-----------------|
| 0 로그인 | ⚠️ 최소화 | Mock 진입으로 대체 가능 |
| 1 브리핑 | ✅ 필수 | 첫 인상 = 데모 분위기 결정 |
| 2 한 줄 입력 | ✅ 필수 | 핵심 UX |
| 3 병렬 실행 표시 | ✅ 필수 | "AI 팀" 느낌의 핵심 |
| 4 결과물 카드 | ✅ 필수 | 구체적 가치 전달 |
| 5 원클릭 승인 | ✅ 필수 | Level 1 제로휴먼 증명 |
| 6 모드 전환 | ⚡ 보너스 | 확장성 어필 (없어도 됨) |

## 데모 블로커 (이게 안 되면 데모 불가)

```
❶ 오케스트레이터가 한 줄 지시를 해석하고
   에이전트 2개 이상에게 병렬 위임 → 응답 수집

❷ Complaint Agent가 민원 원문 → 초안 출력

❸ Retention Agent가 출결 데이터 → 이탈 위험 학생 목록 출력

❹ 결과물이 카드 UI로 표시되고 "승인" 클릭 가능
```

이 4개 외의 모든 기능은 v2 또는 데모 이후.

## Mock 데이터 설계 (데모 시나리오)

### 학원 (원장 시나리오)

**민원 3건**
```json
[
  { "id": 1, "from": "홍길동 어머니", "content": "이번 달 성적이 왜 이렇게 안 좋나요", "time": "어제 22:14" },
  { "id": 2, "from": "박민준 어머니", "content": "다음 주 수업 보강 가능한가요", "time": "오늘 07:32" },
  { "id": 3, "from": "김서연 아버지", "content": "수업 중 선생님이 아이한테 너무 엄하다고 합니다", "time": "오늘 08:51" }
]
```

**출결 데이터 (이탈 위험)**
```json
[
  { "name": "박민준", "absences_this_month": 4, "pattern": "월요일 반복 결석", "risk": "HIGH" },
  { "name": "이수아", "absences_this_month": 2, "trend": "지난달 대비 2배", "risk": "MEDIUM" }
]
```

**에이전트 출력 예시 (Complaint Agent)**
```
민원 #1 — 홍길동 어머니
분류: 성적 불만 | 긴급도: 보통 | 에스컬레이션: 불필요

[응답 초안]
"안녕하세요 어머니, 홍길동 학생의 이번 달 성취도를 함께 검토해보겠습니다.
최근 수업 참여도와 과제 완성률을 기반으로 분석해보면...
다음 주 상담 일정을 잡아보시겠어요? 감사합니다."
```

### 학교 (교사 시나리오, 확장 장면용)

**민원 1건**
```json
{ "from": "홍길동 어머니", "content": "수업 중 목소리가 너무 커서 아이가 잠을 못 잡니다", "time": "어제 23:40" }
```

## 개발 완료 기준 (Definition of Done)

- [ ] 한 줄 입력 → 2개 에이전트 병렬 호출 → 결과 수집 (백엔드)
- [ ] 에이전트 실행 중 상태가 UI에 실시간 표시됨
- [ ] 민원 초안 카드가 읽을 수 있는 형태로 렌더링됨
- [ ] 이탈 위험 학생 카드가 이름·사유와 함께 출력됨
- [ ] "승인" 버튼 클릭 시 상태 변경 확인
- [ ] Mock 데이터 기반 전체 흐름이 끊김 없이 2분 내 완료
