# Problem Scorecard

> 5점 척도. 🟢 = 현장 근거 확보 | 🟡 = 유사 근거 | 🔴 = 가설
> 총점 18점 이상이면 MVP 후보로 진행.

---

## 채점 기준

| 항목 | 1점 | 3점 | 5점 |
|------|-----|-----|-----|
| **현장성** | 논리로만 추정 | 유사 사례·기사 있음 | 현장 발화·공신력 있는 통계 직접 확인 |
| **AI 적합성** | AI 없이도 됨 | AI가 도움되지만 단일 챗봇으로 가능 | 역할 기반 agent orchestration 없이 해결 불가 |
| **7일 MVP** | 기술 스택·범위 불명확 | 구현 가능하지만 범위 조정 필요 | 지금 당장 시작 가능, 데모 가능 |
| **데모 전달력** | 결과가 비직관적 | 설명이 있으면 이해 가능 | 2분 안에 "와" 소리 나는 장면 있음 |
| **확장성** | 이 사용자군 외에 없음 | 한 축만 확장 가능 | 복수 축(사용자/도메인/기능)으로 확장 가능 |

## 항목별 채점 근거

### P-001 학원 원장 운영 보조

| 항목 | 점수 | 근거 |
|------|------|------|
| 현장성 | **4** 🟢 | bati.ai 분석: 원장 행정 하루 3시간 = 월 300만원 손실. 클래스업/온하이 기능 목록으로 미커버 영역 확인 |
| AI 적합성 | **5** 🟢 | 민원 분류·이탈 예측·강사 관리·보고서 생성이 동시에 발생 → 단일 챗봇 불가, agent 분담 필수 |
| 7일 MVP | **3** 🟡 | Claude API + Supabase + Next.js 조합은 검증됨. 단, 2~3개 agent 협업 구현 범위 조정 필요 |
| 데모 전달력 | **4** 🟡 | "원장 1명 지시 → 3개 업무 동시 처리" 직관적. KEG 심사위원(교육그룹 IT) 직접 공감 가능. mock 데이터 퀄리티에 의존 |
| 확장성 | **5** 🟢 | 사교육 원장 → 공교육 교장 → 기업 교육 운영자. KEG 7개 지점 다지점 확장 시나리오 |

### P-002 담임교사 반 운영

| 항목 | 점수 | 근거 |
|------|------|------|
| 현장성 | **5** 🟢 | OECD TALIS 2024: 행정업무 주 6시간 (OECD 평균 2배, 세계 1위). 민원 스트레스 56.9% (세계 2위). 공신력 있는 통계 |
| AI 적합성 | **4** 🟢 | 민원 초안·수업계획·학생 요약을 agent 분담 처리. 생기부 단독은 경쟁 있음 → 민원+수업계획 조합으로 AI 필요성 명확 |
| 7일 MVP | **4** 🟡 | 외부 시스템 연동 불필요 (NEIS 연동 없이 입력 기반 생성만으로도 데모 가능). 단, mock 학생 데이터 설계 필요 |
| 데모 전달력 | **5** 🟢 | "교사 한 줄 → 민원 초안 + 수업 계획 즉시 출력" 비개발자도 즉각 이해. 교육 현장 공감대 큼 |
| 확장성 | **3** 🟡 | 공교육 전체 교사층 확장 가능. 단, 공교육 규제(개인정보, NEIS)가 사교육보다 속도 제한 |

### P-003 학원 신규 상담 자동화

| 항목 | 점수 | 근거 |
|------|------|------|
| 현장성 | **3** 🟡 | bati.ai에서 신규 상담 자동화 필요성 언급. 정량적 이탈률 데이터는 미확보 |
| AI 적합성 | **4** 🟡 | 24시간 문의 응대 + 레벨 진단 + 반 추천은 agent chain으로 설득력 있음 |
| 7일 MVP | **3** 🟡 | 카카오채널 연동이 변수. 웹 UI만으로 데모하면 임팩트 약해짐 |
| 데모 전달력 | **4** 🟡 | "문의 → 레벨 진단 → 반 추천" 흐름이 직관적 |
| 확장성 | **4** 🟡 | 학원 유형 불문 적용 가능 |
