## R-001. 교강사 업무 페인포인트

```text
너는 에듀테크 리서처다. 한국의 직업교육/학원/대학 교강사들이 일상적으로 겪는 업무 페인포인트를 깊이 조사하라.

조사 범위:
- 수업 준비 (교안 작성, 자료 제작, 커리큘럼 설계)
- 수업 진행 (출결, 학습자 참여, 실시간 피드백)
- 수업 후 (과제 채점, 피드백 작성, 학습 성과 분석)
- 행정 업무 (수강생 관리, 성적 보고, 상급자 리포팅)
- 학습자 코칭 (개인별 수준 파악, 진도 조절, 동기부여)
- 교강사 간 협업 (교안 공유, 노하우 전수, 강의 품질 표준화)

각 페인포인트에 대해:
1. 구체적 상황 묘사 (누가, 언제, 어디서, 무엇을 하다가)
2. 현재 해결 방식과 그 한계
3. AI로 해결할 수 있는 가능성 (높음/중간/낮음 + 이유)
4. 이 문제를 7일 안에 MVP로 만들 수 있는지 (가능/어려움 + 이유)

출력: 최소 15개 페인포인트. 표 형식.
```

## R-002. 수강생 학습 경험 페인포인트

```text
너는 학습 경험(LX) 연구자다. 한국의 직업교육/IT학원/온라인 교육에서 수강생들이 겪는 학습 페인포인트를 조사하라.

조사 범위:
- 학습 시작 전 (과정 선택, 수준 파악, 학습 로드맵 부재)
- 학습 중 (이해도 차이, 질문 장벽, 실습 환경, 동기 저하)
- 학습 후 (복습, 포트폴리오 정리, 취업 연계, 역량 증명)
- 동료 학습 (스터디 그룹, 코드 리뷰, 프로젝트 협업)
- 자기주도 학습 (진도 관리, 학습 습관, 번아웃)
- 평가/피드백 (과제 피드백 지연, 성적 불투명, 개선 방향 부재)

각 페인포인트에 대해:
1. "나는 ___할 때 ___해서 ___가 어렵다" 형식의 문제 문장
2. 현재 대안과 불만족 이유
3. AI가 어떻게 도울 수 있는가 (구체적 시나리오)
4. 기존 에듀테크 서비스 중 이 문제를 시도한 사례와 한계

출력: 최소 15개 페인포인트. 우선순위 순으로 정렬.
```

## R-003. 교육 운영/행정 페인포인트

```text
너는 교육 운영 컨설턴트다. 한국의 학원/직업교육기관/대학의 교육 운영자(행정, 상담, 취업지원)가 겪는 업무 페인포인트를 조사하라.

조사 범위:
- 상담 업무 (입학 상담, 과정 추천, 반복 질의 응대, 상담 기록)
- 수강 관리 (수강 신청, 반 편성, 출결 관리, 중도 이탈 예측)
- 취업 지원 (이력서 첨삭, 포트폴리오 정리, 면접 준비, 기업 매칭)
- 기관 운영 (강사 스케줄링, 강의실 배정, 교재/자원 관리)
- 성과 리포팅 (수료율, 취업률, 만족도, 상급 기관 보고)
- 학부모/기업 소통 (학습 현황 공유, 성과 리포트, 불만 처리)

각 페인포인트에 대해:
1. 소요 시간/빈도 (매일 X시간, 주 Y회 등)
2. 반복적이고 자동화 가능한 부분
3. AI 적용 시 예상 효과 (시간 절감, 품질 향상, 이탈 방지 등)
4. 민감 데이터/개인정보 이슈 여부

출력: 최소 12개 페인포인트. 자동화 가능성 높은 순으로 정렬.
```

## R-004. 에듀테크 시장 경쟁 분석

```text
너는 에듀테크 시장 분석가다. 2024~2026년 한국 에듀테크 시장의 현황과 트렌드를 분석하라.

분석 항목:
1. 시장 규모와 성장률 (국내/글로벌)
2. 주요 플레이어 카테고리별 정리:
   - LMS/학습관리: 클래스팅, 구름, Canvas, 에드몬드 등
   - AI 튜터/코칭: 뤼이드(산타), 매스프레소(콴다), Duolingo 등
   - 코딩 교육: 엘리스, 코드잇, 프로그래머스, 인프런 등
   - 직업교육/취업: 패스트캠퍼스, 스파르타코딩, 제로베이스 등
   - AI 기반 평가: Turnitin, 코드그레이더, 자동 채점 도구 등
3. 아직 제대로 해결되지 않은 영역 (화이트스페이스)
4. 최근 투자/인수 동향에서 보이는 방향성
5. 규제/정책 동향 (교육부 AI 정책, 디지털교과서, 공교육 AI)

출력: 카테고리별 경쟁 맵 + 기회 영역 3~5개 제안.
```

## R-005. AI × 교육 기술 트렌드

```text
너는 AI 교육 기술 연구자다. 2024~2026년 AI가 교육에 적용되는 최신 기술 트렌드를 조사하라.

조사 영역:
1. 생성형 AI 교육 적용 (GPT 기반 튜터, 교안 자동생성, 코드리뷰 봇)
2. 적응형 학습 (Adaptive Learning) — 개인화 학습 경로
3. 자연어 기반 평가/피드백 (에세이 채점, 코드 피드백, 구술 평가)
4. 음성/영상 AI (강의 요약, 실시간 번역, 발표 피드백)
5. 학습 분석(Learning Analytics) — 이탈 예측, 참여도 분석
6. AI 에이전트 (자율적으로 학습 계획을 세우고 조정하는 시스템)
7. 게이미피케이션 + AI (적응형 난이도, 개인화 보상)
8. RAG 기반 교육 (교재/강의자료를 기반으로 한 지능형 Q&A)

각 트렌드에 대해:
1. 기술 성숙도 (실험/초기 상용/대중화)
2. 7일 안에 MVP로 구현 가능한 형태
3. 대표 사례 (서비스명, 기업명)
4. 한국 교육 현장 적용 시 장벽

출력: 트렌드별 요약 + "7일 MVP 가능" 여부 표시.
```

## R-006. 직업교육/학원 특화 문제

```text
너는 한국 직업교육 전문가다. 대회 주최인 코리아IT아카데미와 같은 민간 직업교육기관(IT학원, 코딩부트캠프)이 직면한 고유 문제를 조사하라.

이 기관들의 특성:
- 6개월~1년 과정, 비전공자 대상 IT 직무 교육
- 수료 후 취업 연계가 핵심 KPI
- 강사 순환이 빠르고, 교안 표준화가 어려움
- 수강생 수준 편차가 크고, 중도 이탈률이 높음
- NCS(국가직무능력표준) 기반 과정 운영
- 국비지원 과정의 경우 HRD-Net 보고 의무

조사 항목:
1. 강사 관점: 비전공자 교육의 어려움, 수준별 수업, 교안 재사용
2. 운영자 관점: 중도 탈락 관리, 취업률 관리, HRD 보고, 상담 과부하
3. 수강생 관점: 진도 따라가기, 포트폴리오 완성, 취업 불안
4. 기관 관점: 강사 품질 관리, 커리큘럼 업데이트 주기, 경쟁기관 대비 차별화

각 문제에 대해:
1. 문제의 심각도 (1~5)
2. AI 솔루션 적합도 (1~5)
3. 7일 MVP 구현 가능성 (1~5)
4. 대회 심사 어필 가능성 (1~5)

출력: 문제 우선순위 매트릭스.
```

## R-007. 교육 UX/서비스 디자인 패턴

```text
너는 교육 UX 디자이너다. 교육 서비스에서 검증된 UX 패턴과 디자인 원칙을 조사하라.

조사 범위:
1. 온보딩: 수준 진단 → 맞춤 학습경로 추천 플로우
2. 학습 대시보드: 진도, 성취, 약점, 추천 콘텐츠 시각화
3. 피드백 루프: 과제 제출 → AI 피드백 → 개선 → 재제출

5. 접근성: 다양한 디바이스, 학습 장애, 다국어
6. 강사 도구: 학습자 현황 대시보드, 개입 시점 알림

각 패턴에 대해:
1. 효과를 입증한 사례 (서비스명)
2. 핵심 원리 (왜 효과적인가)
3. 최소 구현에 필요한 기술 요소
4. 7일 안에 데모로 보여줄 수 있는 수준

출력: 패턴별 카드 형식. "데모 임팩트" 순으로 정렬.
```
